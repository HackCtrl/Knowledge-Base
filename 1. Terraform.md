***Terraform*** - он нужен для разворачивания, изменения, удаление и резервного копирования виртуальных машин и контейнеров в облачных инфраструктурах, системах виртуализации и контейнеризации 

***Составляющие инструменты:*** 
1. ***Исполняемый файл*** 
- Terraform распространяется не как сервис, а как исполняемый файл. И это предельно упрощает установку и использование: достаточно скачать и скопировать его в `/usr/bin` или воспользоваться [инструкцией с официального сайта](https://developer.hashicorp.com/terraform/install#linux).
1. ***Провайдеры*** 
- Это дополнительный модуль с возможностями для управления конкретным вендором (облачной инфраструктурой, системой виртуализации или контейнеризации) вроде Яндекс Облака.
1. ***Конфигурационные файлы*** 
- В них описывают переменные, настройки для провайдера, виртуальные машины и контейнеры, а ещё указывают, что должно отображаться в консоли после выполнения задачи
## ***HCL и конфигурационные файлы***

Terraform использует собственные объекты и не привязан к отступам: если где-то в одном и том же элементе вы сделали два пробела, а где-то — десять, это не будет ошибкой синтаксиса, как например в yaml.

***Переменные. Для их описания обязательны два файла:*** 
- `variables.tf` - описываются стандартные значения переменных, которые будут использоваться при деплое

***Пример файла:*** 
```
variable "virtual_machines" {
	default = "" 
	
} 

variable "my_variable" { 
	default = "test" 
}
```

- `terraform.tfvars` - можно в свободном виде указать значение переменных, которые есть в variables.tf

*Важный момент: если в `terraform.tfvars` не будет указано значение какой-либо переменной, описанной в `variables.tf`, во время деплоя ВМ будут использованы стандартные значения из `variables.tf`*

***Пример:*** 
```
virtual_machines = { 
	"vm-1" = { 
		name = "some_name" 
		var = "test" 
		disk_size = 20 
	}, 
	"vm-2" = { 
		name = "some_name-etc" 
		var = 1 
		disk_size = 200 
	} 
} 

my_variable = "string"
```
### ***main.tf***

***`main.tf`*** - второй по важности файл. В нем мы описываем желаемый результат. Для этого используются сущности `data` и `resource`

***Синтаксис:*** 
```
# Общий вид 
	data "параметр из провайдера" "любое название объекта" { 
<новая переменная> = <значение> 
} 

# Пример для VmWare. Тут мы получим от API объект со всеми значениями конкретного дата-центра, и в этот полученный объект дополнительно запишется переменная "name", описанная в блоке. 
	data "vsphere_datacenter" "dc" { 
name = "${var.vsphere_datacenter}" 
}
```

***`resource`*** - применяется для описания параметров ВМ, дисков, сетевых интерфейсов и так далее, которые будут использоваться при деплое. 

***Синтаксис:*** 
```
resource "параметр из провайдера" "любое название объекта" { 
	<ключ, заложенный в провайдер> = <любое значение> 
	}
```
### ***output.tf***

***`output.tf`*** - описывает, какая информация выведется в консоль после работы с ВМ. Для этого нужно добавить слово `output`, затем назначить ключ для отображения, а уже внутри этого объекта — указать, что будет выведено

***Синтаксис:*** 
```
output "vm_ip" { 
	value = { for k, v in <resource из main.tf>.<название объекта, которое вы указывали в этом ресурсе в main.tf> : k => v.<ключ объекта>.<нужный индекс> } 
}
```

Цикл `for_each` используется в качестве создания нескольких ресурсов в облаке

- `terraform.tfstate` - записывается, что будет выполнено в следующем запуске 
- `terraform.tfstate.lock` и `terraform.tfstate.backup` - текущее состояние инфраструктуры
