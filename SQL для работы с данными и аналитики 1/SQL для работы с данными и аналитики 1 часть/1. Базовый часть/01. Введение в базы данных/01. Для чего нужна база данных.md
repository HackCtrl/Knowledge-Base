Если у компании большой документооборот и прочее, хранить это все в таблицах не совсем удобно. Для этого и придумали БД - каждая таблицы хранить определенную информацию, например данные о сотрудниках. Связать сотрудника с продажей можно по номеру - например по идентификатору сотрудника 

***Базы данных бывают разных видов, в зависимости от способа хранения данных:*** 
- Реляционные (связь) - данные представленные в виде связанных таблиц (особенность - не должно быть дубликатов, элементы носят особое название (записи - ячейки, поля - строки, объекты на пересечение полей - ячейками))

*Уникальный ключ сделает уникальной всю запись* 

***Для чего нужен первичный ключ:*** 
- Первичный ключ предоставляет доступ пользователя к БД 
- Первичный ключ - это просто номер записи 
- Первичный ключ - это система, которая позволяет управлять БД 
- Первичный ключ делает запись уникальной и позволяет избежать полных дубликатов из таблицы
