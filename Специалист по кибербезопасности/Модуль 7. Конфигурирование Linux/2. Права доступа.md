### ***Discretionary access control (DAC)***

***Discretionary access control (DAC)*** - это модель управления доступом в компьютерных системах, при которой владельцы ресурсов (файлов, папок, БД, устройств и так далее) самостоятельно определяют, кто может получить к ним доступ и какие действия с ними можно выполнить 

`whoami` - команда в Linux, которая отображает имя пользователя вошедшего в систему в данный момент

`su -` -  команда для переключения пользователя, `-` - используется для получения полного окружения пользователя 

`id` - это утилита для отображения информации о пользователе и его идентификаторах в системе. Она позволяет узнать **UID** (User Identifier — идентификатор пользователя), **GID** (Group Identifier — идентификатор группы) и **список групп**, к которым принадлежит пользователь

`/etc/passwd` - список всех пользователей 
`/etc/group` - список всех групп, которые есть в линуксе 
`/etc/shadow` - хранить хэши всех пользователей, хэши паролей всех пользователей 

```
-rw-------: Файл, который доступен только его владельцем.
-rwxr-хт-х: Файл, который является исполняемым любым пользователем в системе.
-rw-rw-rw-: Файл, который открыт для модификации каждого пользователя в системе.
drwxr-хт-х: каталог, что каждый пользователь в системе может читать и доступ.
drwxrwx ---: Каталог, который является изменяемым (включая его содержание) его
владельцем и группой.
drwxr-х ---: Каталог, который доступен по своей группе.


t-bit - разрешает удаление файла из каталога только владельцу файла
0 --- ---
1 --x 001
2 -w- 010
3 -wx -11
4 r-- 100
5 r-x 101
6 rw- 110
7 rwx 111
```

***chmod*** - это утилита Unix-подобных систем, которая позволяет изменять права доступа к файлам и каталогам 

`setuid set user ID upon execution` - установка ID пользователя во время выполнения ( разрешают пользователям запускать исполняемые файлы с правами владельца исполняемого файла.) Процесс будет иметь те же права что и владелец файла.
### ***ACL***

***ACL (Access Control List — «списки управления доступом»)*** - это механизм, который позволяет гибко задавать права пользователей и групп по отношению к объектам (файлам, каталогам, сетевым ресурсам и так далее)

```
getfacl test
setfacl -m u:dummy:r /tmp/test_acl
setfacl -x u:iivanov secretfile - удалить права
setfacl -R -x u:apetrov folder удалить рекурсивно все пр
ава в директории
```

- `getfacl` - это утилита в линукс, которая отображает списки контроля доступа для файлов и каталогов

### ***SELINUX***

`SELINUX` - система обеспечивающая мандатную модель контроля доступа 
- ***Мандатное управление доступом (англ. Mandatory access control, MAC)*** - разграничение доступа субъектов к объектам, основанное на назначении метки конфиденциальности для информации, содержащийся в объектах, и выдаче официальных разрешений (допуска) субъектам на обращение к информации такого уровня конфиденциальности 

***Субъект*** - процесс, ***Объект*** - файл или системный вызов. Взаимодействие субъектов и объектов помечаются специальной меткой - контекст безопасности 

- ***SELINUX Policy*** - коллекции правил, которые определяют, какой источник (субъект) должен иметь право к цели (объект)

- ***Source domain*** - объект пытающийся получить доступ к цели. Обычно это процесс или пользователь 

- ***Target domain*** - объект к которому пытаются получить доступ. Обычно файл или порт 

- ***Context*** - метка безопасности, которая используется для категоризации объекта 

- ***Rule*** - специальная часть политики, которая определяет, какой источник имеет права к Target domain

***Состояние SELINUX***
- `enforcing` - блокирует неразрешённую активность 
- `permissive` - только логирует активность, но не блокирует 
- `disabled` - выключено 

`cat /etc/sysconfig/selinux` - посмотреть настройки selinux либо 
`sestatus -v`

 `setenforce 0` - выключить selinux 

`ls -laZ` - буква `Z`, показывает контекст, который приписывает к ним selinux 

```
ps -eZ [| gerp имя_программы] - посмотреть контекст запущенного процесса
ls -Z : контекст SELinux для файлов.
id -Z : контекст SELinux для текущего пользователя.
```
