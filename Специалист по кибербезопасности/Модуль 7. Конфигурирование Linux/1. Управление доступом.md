***Security Content Automation Protocol***  - некая организация, которая занимается автоматизацией комплаинсов безопасностью линукс систем. То есть это набор стандартов и спецификаций, разработанный для автоматизации управления безопасностью ИС. **Его цель** - упростить и ускорить процесс оценки и управления рисками в сфере ИБ, обеспечив автоматизированные и стандартизированные методы анализа безопасности  

***Управление доступом*** - это комплекс процессов, технологий и политик, направленных на контроль того, кто, когда и при каких условиях может получать доступ к информационным ресурсам организации. **Его цель** - минимизировать риски несанкционированного доступа, утечки данных и других угроз, связанных с вмешательством третьих лиц

***Linux Hardering*** - это систематический процесс усиления безопасности ОС Linix путем минимизации ее уязвимостей, удаление ненужных компонентов и настройки конфигурации с учетом лучших практик безопасность. **Его цель** - снизить поверхность атаки, ограничить возможности злоумышленников и повысить устойчивость системы к угрозам  

*Так как данный урок модуля состоит почти из практики, то буду описывать проделанные действия в ходе прохождения курса*
### ***Оценка корректности настройки CentOS***

***OpenSCAP-scanner*** - это инструмент для сканирования уязвимостей и проверки соответствия систем стандартам безопасности, основанной на линейке стандартов SCAP  

***SCAP Security Guide*** - проект с открытым исходным кодом, который предоставляет руководство по настройки систем с точки зрения безопасности в формате SCAP. Он содержит практические рекомендации по усилению защиты систем, связанные с различными стандартами и требованиями 

Данные инструменты я установил на свою виртуальную машину CentOS, дальше исходя из урока буду практиковаться 

`/usr/share/xml/scap/ssg/content/` - это хранилище контента для стандарта SCAP, который используется для автоматизации оценки безопасности систем, управления уязвимостями и проверки соответствия политикам  

`/usr/share/xml/scap/ssg/rhel7/ssg-rhel7-ds.xml` - это файл исходного потока данных из проекта SCAP, который используется в рамках стандарта SCAP для автоматизации проверки соответствия систем требованиям безопасности и поиска уязвимостей

***Каждый такой xml включает в себя:*** 
- ***XCCDF-документ*** - задает структуру чек-листа безопасности: набор правил, групп, профилей и тестов 
- ***OVAL-определения*** - содержит логические проверки для выявления нарушений конфигурации или отсутствия обновления 
- ***OCIL-файлы*** - содержит вопросы для интерактивного опроса, когда автоматически проверка невозможна 
- ***Методы исправления (remediation)*** - скрипты: bash, ansible, puppet и др. автоматически устраняющие найденные отклонения

```
  /usr/share/xml/scap/ssg/content/ssg-rhel10-ds.xml

```

`notapplicable` - статус, обозначает, что правило не применимо к текущей конфигурации
