# Правила разработки DTEK Core

## Основные принципы

1. Не писать код без задачи.
2. Не писать код без документации.
3. Не писать код напрямую в ветку main.
4. Любая функциональность должна быть описана до реализации.
5. Любая задача должна иметь критерии готовности.

## Процесс разработки

`Идея -> Документация -> Задача -> Ветка Git -> Разработка через Claude Code -> Проверка -> Тестирование -> Merge`

## Работа с Git

**Основные ветки:**
- main
- develop

Для каждой задачи создается отдельная ветка:

- feature/auth
- feature/organizations
- feature/objects
- feature/trust-scor

## Работа с Claude Code

**Перед началом работы Claude Code должен получить:**
- описание проекта;
- архитектуру проекта;
- описание задачи;
- критерии готовности задачи.

*Claude Code не должен самостоятельно изменять архитектурные решения проекта. Архитектурные решения принимаются командой проекта.*

## Работа с документацией

Документация является частью продукта. Любая новая функциональность должна сопровождаться обновлением документации. Документация имеет такой же приоритет, как и код.
