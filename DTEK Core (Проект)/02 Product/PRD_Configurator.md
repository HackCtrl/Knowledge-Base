# PRD Configurator

## Назначение

Конфигуратор предназначен для адаптации DTEK Core под конкретную организацию. Пользователь проходит пошаговый опрос. На основе ответов система формирует цифровой профиль организации.

## Цель

- Снизить сложность внедрения платформы. 
- Минимизировать необходимость ручной настройки системы.

## Сценарий работы

**Шаг 1.**
Создание организации.

**Шаг 2.**
Заполнение профиля организации.

**Шаг 3.**
Выбор отрасли.

**Шаг 4.**
Указание масштаба организации.

**Шаг 5.**
Указание используемых объектов.

**Шаг 6.**
Формирование базовой конфигурации DTEK Core.

## Параметры организации MVP

### Общие данные

- название;
- отрасль;
- количество сотрудников;
- количество филиалов.

### Инфраструктура

- серверы;
- рабочие станции;
- ноутбуки;
- мобильные устройства.

### Информационные системы

- веб-сервисы;
- базы данных;
- внутренние приложения.

### Объекты повышенного риска

- транспорт;
- IoT;
- промышленное оборудование;
- беспилотные системы.

## Результат работы

**После завершения конфигурации система автоматически:**
- создает структуру организации;
- создает базовые категории объектов;
- генерирует стартовые паспорта доверия;
- формирует начальный граф доверия.

## Ограничения MVP

- Конфигуратор не создает инфраструктуру автоматически.
- Конфигуратор не выполняет интеграции.
- Конфигуратор используется исключительно для первичной настройки платформы.

## Критерий готовности

Пользователь может пройти конфигуратор и получить полностью готовую рабочую среду DTEK Core.
