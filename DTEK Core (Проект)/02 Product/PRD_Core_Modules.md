# PRD Core Modules

## Модуль 4. Trust Passport

### Назначение

- Trust Passport является цифровым паспортом доверия объекта.
- Каждый объект системы обязан иметь собственный паспорт доверия.
- Паспорт выступает центральной сущностью платформы DTEK Core.

### Цель

Предоставить единое представление об объекте, его характеристиках, рисках, связях и уровне доверия.

### Структура паспорта

#### Общая информация

- ID объекта
- Название
- Тип объекта
- Владелец
- Организация
- Дата создания

#### Оценка доверия

- Trust Score
- Уровень критичности
- Статус доверия

#### Риски

- Список рисков
- Количество рисков
- Критичность рисков

#### Связи

- Связанные объекты
- Количество связей

### Основные функции

- просмотр паспорта;
- обновление паспорта;
- история изменений;
- просмотр связанных объектов.

### Критерий готовности

Пользователь может открыть паспорт объекта и получить полную информацию о нем.

## Модуль 5. Trust Score

### Назначение

Trust Score определяет уровень доверия к объекту.

### Диапазон

От 0 до 100.

### Категории

`0-20` — Критический уровень

`21-40` — Низкий уровень

`41-60` — Средний уровень

`61-80` — Высокий уровень

`81-100` — Максимальный уровень

### Факторы оценки MVP

- критичность объекта;
- количество рисков;
- наличие связей;
- статус объекта;
- полнота заполнения данных.

### Основные функции

- автоматический расчет;
- отображение оценки;
- история изменения оценки.

### Критерий готовности

Пользователь видит рассчитанный уровень доверия объекта.

## Модуль 6. Trust Graph

### Назначение

Визуализация связей между объектами организации.

### Цель

Позволить пользователю понимать взаимосвязи объектов и последствия компрометации.

### Основные функции

- просмотр графа;
- создание связи;
- удаление связи;
- фильтрация объектов;
- поиск объектов.

### Типы связей MVP

- использует;
- принадлежит;
- зависит от;
- управляет;
- взаимодействует.

### Критерий готовности

Пользователь может построить граф связей между объектами организации.

## Модуль 7. Реестр рисков

### Назначение

Учет рисков объектов организации.

### Поля риска

- ID
- Название
- Описание
- Критичность
- Вероятность
- Статус

### Основные функции

- регистрация риска;
- редактирование риска;
- закрытие риска;
- привязка риска к объекту.

### Критерий готовности

Пользователь может зарегистрировать риск и привязать его к объекту.
