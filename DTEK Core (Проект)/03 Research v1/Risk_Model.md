# Risk Model

## Назначение

Risk Model определяет правила учета, хранения и влияния рисков на объекты системы DTEK Core. Риск является одним из основных факторов, влияющих на уровень доверия объекта.

# Цели модели рисков

1. Централизованный учет рисков.
2. Связь рисков с объектами.
3. Влияние рисков на Trust Score.
4. Анализ критичных объектов.
5. Формирование цифрового профиля безопасности организации.

# Определение риска

**Риск** — потенциальное событие или условие, которое может негативно повлиять на объект организации.

# Структура риска

## Основная информация

**Поля:**
- Risk ID
- Название риска
- Описание риска
- Дата создания
- Автор

## Классификация

**Категория риска:**
- Организационный
- Технический
- Информационный
- Физический
- Человеческий фактор

## Критичность

**Значения:**
- Low
- Medium
- High
- Critical

## Вероятность

**Значения:**
- Low
- Medium
- High

## Статус

- Open
- Mitigated
- Closed
- Accepted

# Связь риска с объектом

- Один риск может относиться к нескольким объектам.
- Один объект может иметь несколько рисков.
- Связь реализуется через таблицу Object Risks.

# Влияние на Trust Score

**Low:**
`-2`

**Medium:**
`-5`

**High:**
`-10`

**Critical:**
`-20`

# Жизненный цикл риска

`Создание -> Анализ -> Оценка -> Обработка -> Закрытие`

# Отображение в системе

**Каждый риск должен отображаться:**
- в Trust Passport;
- в карточке объекта;
- в реестре рисков;
- в графе доверия.

# Реестр рисков

Система должна предоставлять единый список рисков организации.

**Для каждого риска отображается:**
- название;
- объект;
- критичность;
- статус;
- влияние на Trust Score.

# Метрики MVP

- Количество открытых рисков.
- Количество критических рисков.
- Количество закрытых рисков.
- Средняя критичность рисков организации.

# Будущее развитие

**В следующих версиях модель может учитывать:**
- уязвимости;
- результаты сканирования безопасности;
- данные SIEM;
- данные EDR;
- результаты аудитов;
- требования регуляторов.
