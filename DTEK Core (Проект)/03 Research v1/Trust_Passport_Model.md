# Trust Passport Model

## Назначение

- Trust Passport является цифровым паспортом доверия объекта.
- Каждый объект DTEK Core обязан иметь собственный паспорт доверия.
- Паспорт является основной единицей хранения информации об объекте.

# Цели Trust Passport

1. Централизованное хранение информации об объекте.
2. Отображение уровня доверия.
3. Отображение рисков.
4. Отображение связей.
5. История изменений объекта.

# Структура паспорта

## Блок 1. Основная информация

**Поля:**
- Passport ID
- Object ID
- Название объекта
- Тип объекта
- Организация
- Владелец
- Дата создания
- Дата последнего изменения

## Блок 2. Trust Score

**Поля:**
- Текущий Trust Score
- Уровень доверия
- Дата расчета
- История изменения оценки

## Блок 3. Критичность

**Поля:**
- Business Criticality
- Security Criticality

**Значения:**
- Low
- Medium
- High
- Critical

## Блок 4. Риски

**Поля:**
- Количество рисков
- Открытые риски
- Закрытые риски
- Максимальная критичность риска

## Блок 5. Связи

**Поля:**
- Количество связей
- Входящие связи
- Исходящие связи

## Блок 6. Состояние объекта

**Статусы:**
- Active
- Warning
- Risk
- Critical
- Archived

# Основные функции

- просмотр паспорта;
- экспорт паспорта;
- история изменений;
- просмотр связей;
- просмотр рисков.

# Принципы

- Каждый объект имеет только один паспорт доверия.
- Паспорт обновляется автоматически при изменении связанных данных.
- Паспорт выступает единой точкой доступа к информации об объекте.
