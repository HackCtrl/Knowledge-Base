# Trust Score Model

## Назначение

Trust Score является числовой оценкой уровня доверия к объекту.

Оценка рассчитывается автоматически на основе характеристик объекта, рисков и взаимосвязей.

**Trust Score используется для:**
- оценки состояния объекта;
- приоритизации рисков;
- визуализации уровня доверия;
- принятия управленческих решений.

# Диапазон оценки

Trust Score принимает значения от 0 до 100.

# Уровни доверия

`81-100` - Высокое доверие

`61-80` - Достаточное доверие

`41-60` - Средний уровень доверия

`21-40` - Низкий уровень доверия

`0-20` - Критическое состояние

# Модель расчета MVP

**Базовая формула:**

`Trust Score = Base Score - Risk Penalty - Criticality Penalty + Completeness Bonus`

# Начальное значение

**Каждый новый объект получает:**
`Base Score = 70`

# Влияние рисков

**Низкий риск:**
- `-2`

**Средний риск:**
- `-5`

**Высокий риск:**
- -10

**Критический риск:**
- `-20`

# Влияние критичности объекта

**Low**
- `0`

**Medium**
- `-3`

**High**
`-7`

**Critical**
`-15`

# Полнота данных

**Если паспорт заполнен более чем на 80%:**
- `+5`

**Если заполнен более чем на 95%:**
- `+10`

# Минимальные ограничения

**Минимальное значение:**
- `0`

**Максимальное значение:**
- `100`

# Пересчет

**Trust Score пересчитывается при:**
- изменении объекта;
- изменении риска;
- изменении связей;
- обновлении паспорта.

# История изменений

Система должна хранить историю изменения Trust Score.

**Для каждой записи:**
- дата;
- старое значение;
- новое значение;
- причина изменения.

# Будущее развитие

**В следующих версиях Trust Score может учитывать:**
- результаты аудитов;
- уязвимости;
- инциденты безопасности;
- данные мониторинга;
- данные внешних систем.
