# Trust Graph Model

## Назначение

Trust Graph представляет организацию в виде графа взаимосвязанных объектов. Каждый объект является узлом графа. Связи между объектами отображаются в виде ребер.

# Цели

**Trust Graph позволяет:**
- визуализировать структуру организации;
- выявлять критические зависимости;
- анализировать влияние рисков;
- оценивать последствия компрометации объектов;
- понимать распространение угроз.

# Базовые сущности

## Узел (Node)

`Любой объект организации.`

**Примеры:**
- сотрудник;
- сервер;
- ноутбук;
- база данных;
- приложение;
- автомобиль;
- дрон;
- IoT устройство.

## Связь (Edge)

`Отношение между объектами.`

# Типы связей MVP

## Uses

`Использует.`

**Пример:**
- Сотрудник → Ноутбук

## Depends On

`Зависит от.`

**Пример:**
- Приложение → База данных

## Connected To

`Подключен к.`

**Пример:**
- Ноутбук → VPN

## Managed By

`Управляется.`

**Пример:**
- Сервер → Администратор

## Owns

`Владеет.`

**Пример:**
- Организация → Сервер

# Свойства узла

**Каждый узел содержит:**
- ID
- Название
- Тип
- Trust Score
- Критичность
- Статус

# Свойства связи

**Каждая связь содержит:**
- ID
- Тип связи
- Источник
- Назначение
- Дата создания

# Цветовая индикация

**Высокое доверие:**
- Зеленый

**Среднее доверие:**
- Желтый

**Низкое доверие:**
- Оранжевый

**Критическое состояние:**
- Красный

# Анализ влияния

**Система должна определять:**
- Какие объекты зависят от выбранного объекта.

**Пример:**

`Сотрудник -> Ноутбук -> VPN -> Сервер -> База данных`

Компрометация ноутбука влияет на всю цепочку.

# Основные функции MVP

- создание узла;
- создание связи;
- удаление связи;
- просмотр графа;
- поиск объектов;
- фильтрация объектов.

# Ограничения MVP

- Автоматический анализ атак отсутствует.
- Автоматическое построение графа отсутствует.
- Все связи создаются пользователем вручную.

# Будущее развитие

**В будущих версиях система может:**
- автоматически строить граф;
- рассчитывать распространение риска;
- моделировать сценарии атак;
- строить карту цифрового доверия организации в реальном времени.
