# User Roles

## Назначение

Документ описывает роли пользователей в системе DTEK Core MVP.

# Принципы ролевой модели

- Каждый пользователь принадлежит организации.
- Каждый пользователь имеет одну основную роль.
- Права определяются ролью.

# Роль Owner

## Описание

- Владелец организации.
- Главный пользователь системы.

## Возможности

- управление организацией;
- управление пользователями;
- управление объектами;
- управление рисками;
- просмотр графа доверия;
- просмотр всех паспортов доверия;
- удаление данных организации;
- настройка конфигурации.

# Роль Security Officer

## Описание

`Специалист по информационной безопасности.`

## Возможности

- создание объектов;
- редактирование объектов;
- создание рисков;
- управление рисками;
- просмотр паспортов доверия;
- просмотр графа доверия;
- анализ Trust Score.

# Роль Administrator

## Описание

`Системный администратор.`

## Возможности

- создание инфраструктурных объектов;
- редактирование объектов;
- просмотр графа;
- просмотр рисков.

# Роль Viewer

## Описание

`Пользователь только для просмотра.`

## Возможности

- просмотр объектов;
- просмотр паспортов;
- просмотр графа доверия;
- просмотр рисков.

# Матрица доступа

**Организация**
- `Owner:` Full
- `Security Officer:` Read
- `Administrator:` Read
- `Viewer:` Read


**Объекты**
- `Owner:` Full
- `Security Officer:` Full
- `Administrator:` Full
- `Viewer:` Read


**Риски**
- `Owner:` Full
- `Security Officer:` Full
- `Administrator:` Read
- `Viewer:` Read


**Паспорта доверия**
- `Owner:` Read
- `Security Officer:` Read
- `Administrator:` Read
- `Viewer:` Read


**Trust Graph**
- `Owner:` Full
- `Security Officer:` Full
- `Administrator:` Read
- `Viewer:` Read
