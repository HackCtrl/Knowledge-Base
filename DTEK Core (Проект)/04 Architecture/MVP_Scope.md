# MVP Scope

## Назначение

Документ определяет границы первой версии DTEK Core.

# Цель MVP

- Подтвердить востребованность концепции цифрового доверия организации.
- Создать рабочую платформу управления объектами, рисками и уровнем доверия.

# Что входит в MVP

## Аутентификация

- регистрация;
- вход;
- выход;
- роли пользователей.

## Организации

- создание организации;
- управление организацией.

## Пользователи

- приглашение пользователей;
- управление ролями.

## Объекты

- создание объектов;
- редактирование объектов;
- удаление объектов.

## Trust Passport

- просмотр паспорта;
- история изменений;
- отображение рисков.

## Trust Score

- расчет оценки;
- отображение оценки;
- история изменений.

## Risk Registry

- создание риска;
- управление рисками;
- привязка к объектам.

## Trust Graph

- создание связей;
- просмотр графа;
- фильтрация графа.

## Configurator

- первичная настройка организации;
- создание базовой структуры.

# Что НЕ входит в MVP

## AI Assistant

**Причина:**
- Высокая сложность.

## Marketplace

**Причина:**
- Не нужен для проверки гипотезы.

## Интеграции

**Причина:**
- Требуют дополнительные ресурсы.

## Автоматическое сканирование

**Причина:**
- Выходит за рамки MVP.

## Собственные агенты

**Причина:**
- Усложняют разработку.

## Enterprise Runtime

**Причина:**
- Следующий этап развития.

# Критерии успешного MVP

**Пользователь может:**
- создать организацию;
- создать объект;
- получить паспорт доверия;
- зарегистрировать риск;
- увидеть Trust Score;
- увидеть Trust Graph.

# Метрики MVP

- Количество организаций.
- Количество пользователей.
- Количество объектов.
- Количество рисков.
- Количество созданных связей.
- Средний Trust Score организации.
