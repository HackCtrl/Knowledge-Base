При анализе логов систем или приложений часто бывает нужно быстро идентифицировать сообщения определенного типа, например, ошибки. Обычно такие сообщения имеют стандартное начало (префикс), такое как `"[ERROR]"`, `"[WARNING]"`, `"[INFO]"`.

Ваша задача — написать программу, которая проверяет, является ли строка лога сообщением об ошибке, основываясь на ее префиксе.

**Цель**

Написать программу на Python, которая:

1. Получает на вход **одну строку** — строку лога.
2. Проверяет, начинается ли эта строка с префикса `"[ERROR]"` (включая квадратные скобки).
3. Выводит на экран слово `True`, если строка начинается с этого префикса.
4. Выводит на экран слово `False`, если строка начинается с чего-то другого (или пустая).

**Пример ввода:**

[ERROR] Failed to connect to database.

**Пример вывода для этого ввода:**

True

**Другой пример ввода:**

[INFO] User logged in successfully.

**Пример вывода для этого ввода:**

False

**Еще пример ввода:**

[ERROR] Connection timed out.

**Пример вывода для этого ввода:**

True

Эта задача познакомит вас с полезным методом строк для проверки их начала.

### ***Теория***

**Методы строк**

У строк в Python есть встроенные "функции", которые применимы только к строкам. Они называются **методами**. Методы вызываются через точку после переменной или самой строки

**Метод `.startswith(prefix)`** - Этот метод проверяет, начинается ли строка с указанной подстроки (префикса). Он возвращает булево значение

**Синтаксис:**
- `строка.startswith(подстрока_для_проверки)`

**Пример:**

```
log_line = "[ERROR] File not found."

is_error = log_line.startswith("[ERROR]") # is_error будет True

print(is_error) # Выведет: True

log_line_info = "[INFO] Process started."

is_error_info = log_line_info.startswith("[ERROR]") # is_error_info будет False

print(is_error_info) # Выведет: False
```

**Важно!** Метод `.startswith()` чувствителен к регистру букв! `"Error"` и `"error"` — это разные префиксы

### ***Решение*** 

```
log_line = input() 
is_error = log_line.startswith("[ERROR]") 
print(is_error)
```
