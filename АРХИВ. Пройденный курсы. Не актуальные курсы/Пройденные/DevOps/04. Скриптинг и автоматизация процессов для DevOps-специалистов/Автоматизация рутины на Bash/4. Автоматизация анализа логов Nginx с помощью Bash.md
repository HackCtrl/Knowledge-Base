## ***Задание №2. Изучение структуры логов Nginx***

***Разберем, что означают поля в каждой строчке лога:*** 
```
45.33.21.289 - - [01/Jan/2025:08:44:27 +0000] "GET /api/not-extended HTTP/1.1" 510 178 "-" "curl/7.68.0"
```

1. ***IP-адрес клиента (`45.33.21.289`)*** - IP-адрес пользователя или клиентского устройства (или прокси сервера, если он стоит перед Nginx), который сделал запрос 
2. ***Первый дефис (`–`)*** - устаревший идентификатор, полученный по протоколу identd на клиентской машине. Он практически не используется, поэтому в 99,9% случаев мы будем видеть дефис 
3. ***Второй дефис (`–`)*** - имя пользователя, если запрос был к защищенной паролем области (HTTP Basic Auth). Если аутентификация не требуется, это поле остается пустым и обозначается дефисом 
4. ***Временная метка (`[01/Jan/2025:08:44:27 +0000]`)*** - дата и время, когда запрос был обработан сервером: 
	- `01/Jan/2025` - дата/месяц/год 
	- `08:44:27` - часы/минуты/секунды 
	- `+0000` - часовой пояс сервера 
5. ***HTTP-запрос (`"GET /api/not-extended HTTP/1.1"`)*** - составное поле, которое делится на 3 части: 
	- `GET` - HTTP-метод (другие: POST, PUT, DELETE, HEAD, OPTIONS)
	- `/api/not-extended` - путь (url) к запрашиваемому ресурсу на сервер 
	- `HTTP/1.1` - используемая версия HTTP-протокола (также может быть HTTP/2.0, HTTP/1.0)
6. ***Код ответа (`510`)*** - цифровой код, который сервер возвращает клиенту: 
	- `200` - ОК (успешно)
	- `301/302` - перенаправление 
	- `404` - страница не найдена 
	- `500` - внутренняя ошибка сервера 
	- `510` - «Not Extended» (кастомный код для примера)
7. ***Размер ответа (`178`)*** - размер тела ответа сервера клиенту в байтах (без заголовков)
8. ***Referer (`"-"`)*** - URL-адрес веб-страницы, с которой был сделан запрос. Если пользователь перешел по прямой ссылке или ввел URL вручную, это поле будет пустым 
9. ***User-Agent (`"curl/7.68.0"`)*** - строка, которую передает клиент для идентификации себя. Содержит информацию о браузере , его версии, ОС

## ***Задание №4. Создание скрипта анализа логов***

Теперь ваша задача состоит в том, чтобы создать скрипт `analyze_logs.sh`, который будет автоматически анализировать логи и выводить полезную статистику.

1. Создайте скрипт, который принимает имя файла лога в качестве аргумента.
2. Скрипт должен выводить следующую информацию:
    - общее количество запросов,
    - количество уникальных IP-адресов,
    - статистику по кодам ответа,
    - топ-10 IP-адресов по количеству запросов.
3. Добавьте в скрипт проверку существования файла лога.
4. Сделайте скрипт исполняемым:

```
chmod +x analyze_logs.sh 
```

5. Проверьте работу скрипта:

```
./analyze_logs.sh access.log 
```

*Скрипт должен быть универсальным и работать с любым файлом логов nginx в стандартном формате.*

***Решение:*** 

```
#!/bin/bash

# Проверка существования файла лога
if [ ! -f "$1" ]; then
    echo "Ошибка: Файл $1 не найден"
    exit 1
fi

echo "=== Анализ лог-файла: $1 ==="
echo ""

# 1. Общее количество запросов
echo "1. Общее количество запросов:"
wc -l "$1"
echo ""

# 2. Количество уникальных IP-адресов
echo "2. Количество уникальных IP-адресов:"
awk '{print $1}' "$1" | sort | uniq | wc -l
echo ""

# 3. Статистика по кодам ответа
echo "3. Статистика по кодам ответа:"
awk '{print $9}' "$1" | sort | uniq -c
echo ""

# 4. Топ-10 IP-адресов по количеству запросов
echo "4. Топ-10 IP-адресов по количеству запросов:"
awk '{print $1}' "$1" | sort | uniq -c | sort -nr | head -10
```
