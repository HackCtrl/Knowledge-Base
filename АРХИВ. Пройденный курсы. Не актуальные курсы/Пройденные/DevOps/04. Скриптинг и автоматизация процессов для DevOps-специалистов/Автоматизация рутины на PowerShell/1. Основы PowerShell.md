## ***PowerShell и его отличие от Bash***

***PowerShell*** - это одновременно и командная оболочка, и язык сценариев. Он работает не с простыми строками, как Bash, а с объектами. PowerShell обращается к определенному объекту, например к реестру, и в результате выполнения команды отдает обработанный или измененный объект, из которого можно выбрать нужное значение 

***Объект*** - это тип данных, который содержит в себе наборы данных разных типов 

***Пример объекта по аналогии с машиной:*** 

```
Машина:{ #Объект
    Количество колёс: 4, #Значение типа "int (число)"
    Едет: да/нет, #Значение типа "bool (true/false)"
    Части: [капот, двигатель, колеса, двери], #Значение типа "array (массив)"
    Модель: "awesome", #Значение типа "string (строка)"
    Компоненты: { #Объект
        Капот: { #Объект
            Цвет: "красный", #Значение типа "string (строка)"
            Форма: "необычная" #Значение типа "string (строка)"
        }
    }
  }
```

*Объект может показаться похожим на массив. Разница в том, что массив представляет собой набор простых индексированных элементов "ключ - значение", а объект имеет элемент, в который можно вложить дополнительные объекты или иные типы. Объекты помогают удобно передавать и обрабатывать большие и сложные структуры данных* 

Представим, что нам надо получить из всех установленных программ одну конкретную и узнать её версию. Проведём аналогию с Bash на дистрибутиве Ubuntu.

Так как Bash работает со строками, то можно использовать `grep <name>`, чтобы выбрать нужную строку из вывода результата. Команда будет выглядеть так: `dpkg -l | grep <name>`. С PowerShell всё реализуется по-другому. В результате выполнения команды мы получаем не строку, а целый объект, из которого нам необходимо выбрать интересующую программу. Команда может выглядеть так:`Get-ItemProperty HKLM:\\Software\\Microsoft\\Windows\\CurrentVersion\\Uninstall\\* | Where-Object {$_.DisplayName -like "*<name>*"} | Select-Object DisplayName, DisplayVersion, Publisher`

***Здесь мы:*** 
- Обращаемся к реестру Windows, из которого получаем объект с большим количеством значений 
- С помощью `Where-Object {$_.DisplayName -like "*<name>*"}` указываем имя интересующей программы, а с помощью `Select-Object` указываем необходимые поля 
### ***Особенности логики PowerShell***

***Плюсы и минусы:*** 

***Синтаксис:*** 
- Плюсы - Чтобы получить определённый параметр, не надо обрабатывать результат с помощью `sed`, `cut`, `awk`, как в Bash. Достаточно просто сделать выборку с помощью `Select` и `Where-Object` и параметра в объекте
- Минусы - Чтобы получить определённый параметр, не надо обрабатывать результат с помощью `sed`, `cut`, `awk`, как в Bash. Достаточно просто сделать выборку с помощью `Select` и `Where-Object` и параметра в объекте

***Возможности и навыки администратора:*** 
- Плюсы - Поскольку используются разные типы данных, администратор может выбирать тот тип, который займёт меньше оперативной памяти, что сэкономит ресурсы компьютера. К тому же использование разных типов данных упрощает написание скриптов. Например, исключаются проблемы, когда администратор пытается вместо чисел сложить строки
- Минусы - Администратор должен не только понимать работу и структуру скриптов, но и знать большое количество командлетов и уметь делать выборки, используя конвейеры. Более высокий «порог входа» для написания скриптов по сравнению с Bash

***Возможности оболочки:*** 
- Плюсы - Использование объектов и огромное количество командлетов даёт больше возможностей для автоматизации задач и унификации скриптов между различными операционными системами
- Минусы - Важно следить за версией PowerShell. В новых версиях часто изменяют или удаляют функции, из-за чего после обновления системы или интерпретатора скрипты могут не отрабатывать команды и возвращать ошибку

## ***Роль и командлеты PowerShell***

***PowerShell*** - это язык сценариев для автоматизации административных задач в ОС, включая установку программ, управление компонентами и настройку параметров 

***В PowerShell есть командлеты*** - инструкции, который выполняют заложенный в них функционал: 

- `Get-ChildItem` - просмотр содержимого каталога 
- `Add-Content, Clear-Content, Get-Content, Set-Content` - получение/обновление/изменение объекта 
- `Clear-Item, Copy-Item, Get-Item, Invoke-Item, Move-Item, New-Item, Remove-Item, Rename-Item, Set-Item` - получение/обновление/изменение элемента 
- `Clear-ItemProperty`, `Copy-ItemProperty`, `Get-ItemProperty`, `Move-ItemProperty`, `New-ItemProperty`, `Remove-ItemProperty`, `Rename-ItemProperty`, `Set-ItemProperty` - получение/обновление/изменение свойств элемента 
- `Get-Location`, `Pop-Location`, `Push-Location`, `Set-Location` - перемещение по ФС системы 
- `Join-Path`, `Convert-Path`, `Split-Path`, `Resolve-Path`, `Test-Path` - получение/обновление/изменение/объединение путей каталогов файлов 
- `Get-PSDrive`, `New-PSDrive`, `Remove-PSDrive` - работа с дисками 
- `Get-PSProvider` - получение информации о хранилище в текущем сеансе 

***Как будут выглядеть команды, которые мы использовали в уроке с Bash, и результаты их выполнения:*** 

1. ***Ищем только файлы в каталоге `/usr/lib`:*** 
```
Get-ChildItem /usr/lib -File
```

2. ***Посчитаем количество файлов в катологе `/usr/lib`:*** 
```
(Get-ChildItem /usr/lib -File).count
```

![[Pasted image 20260323204136.png]]

`Count` - подсчитывает количество элементов в объекте 

3. ***Создаем каталог, копируем файл и удаляем каталог вместе с файлом:*** 
```
New-Item -Path "/tmp" -Name "practicum" -ItemType Directory #Создадим каталог Copy-Item "/etc/crontab" -Destination "/tmp/practicum" #Скопируем файл Remove-Item /tmp/practicum #Удалим каталог с файлом
```

![[Pasted image 20260323204344.png]]

***Чтобы удалить каталог, в котором есть содержимое, модернизируем команду добавим ключ `Recurse`:*** 
```
Remove-Item /tmp/practicum -Recurse
```

4. ***Посмотрим информацию о дисках и выберем нужный:***
