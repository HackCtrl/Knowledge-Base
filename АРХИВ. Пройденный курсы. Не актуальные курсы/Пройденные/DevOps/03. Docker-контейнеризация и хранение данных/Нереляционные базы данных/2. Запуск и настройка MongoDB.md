***MongoDB Community Server:***
- Бесплатная версия с открытым исходным кодом 
- Подходит для разработки, тестирования и небольших производственных сред 
- Обладает всеми основными функциями MongoDB, включая шардинг, репликацию и индексирование 
- Ограниченная техническая поддержка 

***MongoDB Enterprise Server:***
- Коммерческая версия с расширенными функциями и поддержкой 
- Подходит для предприятий с критичными для бизнеса приложениями 
- Включает в себя все функции MongoDB Community Server, а так же расширенный контроль доступа и аудит 
- Резервное копирование и восстановление с точностью до конкретного момента времени 
- Мониторинг и оповещение в режиме реального времени 
- Приоритетная техническая поддержка 

***MongoDB Atlas:***
- Полностью управляемая версия MongoDB в облаке 
- Удобная, масштабируемая и безопасная 
- Подходит для предприятий, которые хотят сосредоточиться на своих приложениях, а не на управлении БД

***MongoDB Ops Manager:***
- Инструмент управления, который упрощает развертывание, мониторинг и управление кластерами MongoDB 
- Поддерживает как MongoDB Community Server, так и ongoDB Enterprise Server

***Другие версии:***
- ***MongoDB Express:*** легковесное издание для разработчиков и студентов 
- ***MongoDB Stitch:*** бес серверная платформа для создания полнофункциональных приложений MongoDB

*Я буду пользоваться в основном на виндоус ПО* 

*Но установлю данную БД на кали линукс*

***Шаг 1. Добавляем репозиторий MongoDB.***

```
sudo apt-get install gnupg curl sudo curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | \ sudo gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg \ --dearmor
```

***Добавим репозиторий:*** 

```
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
```

***Обновим список пакетов:*** 

```
sudo apt update
```

***Шаг 2. Устанавливаем MongoDB.***

```
sudo apt install -y mongodb-org
```

***Шаг 3. Проверяем статус MongoDB.***

```
sudo systemctl status mongod
```

***Однако не всегда установка происходит успешно*** 

***Одна из распространённых ошибок — `Mongodb exited with code 14`:***

***Исправление (Нужно перенастроить права):***
```
chown -R mongodb:mongodb /var/lib/mongodb 
chown mongodb:mongodb /tmp/mongodb-27017.sock
```

***Разберём компоненты:***
`chown -R mongodb:mongodb /var/lib/mongodb` меняем владельца и группу для всех файлов и директорий в директории `/var/lib/mongodb` на `mongodb`
- `R` означает рекурсивное изменение владельца для всех поддиректорий и файлов в `/var/lib/mongodb`
- `mongodb:mongodb` указывает, что владелец меняется на пользователям `mongodb`, а группа на `mongodb`
- `/var/lib/mongodb` - путь к директории, для которого мы изменяем владельца 

`chown mongodb:mongodb /tmp/mongodb-27017.sock` меняет владельца и группу для файла `/tmp/mongodb-27017.sock`
- `mongodb:mongodb` указывает, что владелец и группа файлов заменяются на пользователя `mongodb` и группу `mongodb`

***Шаг 4. Начинаем работу с MongoDB.***

```
mongosh
```
## ***Основные команды***

1. ***Посмотреть доступные БД:*** 
```
show dbs
```

2. ***Создать БД:*** 
```
use mydb
```

3. ***Создать новую коллекцию в БД - с помощью метода createCollection***
```
db.createCollection("mycollection")
```

4. ***Просмотреть список коллекцией в текущей БД:*** 
```
show collections 
```

5. ***Вставить документы в коллекцию - через `insertOne` или `insertMany`:*** 
```
db.mycollection.insertOne({key: "value"})
```

6. ***Просмотреть содержимое коллекции:*** 
```
db.mycollection.find()
```

7. ***Удалить коллекции:*** 
```
db.mycollection.drop()
```

8. ***Удалить БД***
```
db. dropDatabase()
```

9. ***Обновить документ в коллекцию по определенным критериям:*** 
```
db.название_коллекции.updateOne({ критерий }, { $set: { поле: новое_значение } })
```

10. ***Обновить несколько документов сразу по определённым критериям:*** 
```
db.название_коллекции.updateMany({ критерий }, { $set: { поле: новое_значение } })
```

11. ***Создать индекс в коллекции:*** 
```
db.название_коллекции.createIndex({ поле: 1 })
```

12. ***Просмотреть все индексы коллекции:*** 
- ```
  db.название_коллекции.getIndexes()
  ```

12. ***Удалить индекс и коллекции:*** 
```
db.название_коллекции.dropIndex("название_индекса")
```
## ***Настройки сети***

***Разберём параметры сети MongoDB в файле конфигурации `/etc/mongod.conf`:***
- `net.bindIp` — один или несколько IP-адресов, на которых будет прослушивать MongoDB. По умолчанию это `127.0.0.1`, локальный хост.
- `net.port` — порт, на котором будет прослушивать MongoDB. Значение по умолчанию здесь — `27017`.
- `net.maxIncomingConnections` — максимальное количество одновременных входящих соединений. По умолчанию их `100`.
- `net.maxConnections` — максимальное количество одновременных соединений, включая входящие и исходящие. В этом случае значение по умолчанию — `-1`, то есть без ограничений.
## ***Настройки безопасности***

1. ***Зайти в командную оболочку*** 
```
mongosh
```

2. ***Подключиться к БД admin*** 
```
use admin
```

3. ***Создать пользователя под которым будете заходить:*** 
```
db.createUser({user:"Some_Admin_Account", pwd:passwordPrompt(), roles:[{role:"userAdminAnyDatabase", db:"admin"}, "readWriteAnyDatabase"]})
```

4. ***Теперь включим аутентификацию. Для этого отредактируйте конфигурационный файл `/etc/mongod.conf` и установите параметр `security.authorization` в значение `enabled`:***
```
security: authorization: "enabled"
```

***Изменив параметры перезапустить:*** 
```
systemctl restart mongod
```
