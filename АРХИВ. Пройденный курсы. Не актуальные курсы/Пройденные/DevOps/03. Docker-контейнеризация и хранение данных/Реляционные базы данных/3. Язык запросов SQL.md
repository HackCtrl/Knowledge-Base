***SQL*** - самый распространённый язык запросов для работы с данными, появившийся в результате разработки реляционных БД. Декларативный язык, он описывает результат без алгоритма действия 

*Если базу данных MySQL перенести под управление PostgreSQL, все ваши запросы, написанные в соответствии с SQL-стандартом, будут работать — менять их не придётся*

*Минус данного подхода — в потере гибкости, поэтому зачастую помимо SQL в СУБД поддерживается свой язык запросов, оптимизированный конкретно под неё. Для PostgreSQL это PL/pgSQL, для Oracle — PL/SQL, для MSSQL — Transact-SQL. При этом СУБД может поддерживать сразу несколько языков запросов: это могут быть как расширения SQL, так и другие языки*

***SQL-программа состоит из последовательности команд. Их можно разделить на группы:*** 
- Команда описания структур данных 
- Команды работы с данными 
- Команды управления привилегиями 
- Команды администрирования

`SELECT * FROM pg_roles WHERE rolname='sysadmin';`

*Он означает «выбрать (`select`) записи, включая все столбцы (`*`) из (`from`) таблицы `pg_roles`, у которых (`where`) столбец `rolename` равен `'sysadmin'`»*

*У каждой SQL-команды - свой синтаксис, и каждая оканчивается символом `;`*

***Команды можно:*** 
- Записать в текстовый файл и затем выполнить его с помощью `psql -i my.sql`
- Передать через параметр `psql -с "SQL_COMMAND"`
- Выполнить в интерактивном режиме, подключившись к СУБД через `psql` или другие программы с подобными функциями 
## ***Команды описания структуры данных***

***Сразу пару команд:*** 
- Создать БД `sysadmins_db`: `CREATE DATABASE sysadmins_db;`
- Подключиться к БД для дальнейших команд: `\c sysadmins_db;`

***В пример приведем создание таблиц из первого урока на языке SQL:*** 

- `CREATE TABLE courses (ID integer, Название varchar(200) NOT NULL UNIQUE);` 
- `CREATE TABLE students (ID integer PRIMARY KEY, ФИО varchar(100) NOT NULL CHECK (ФИО <> ''), Курс integer);`

*Для создания таблицы мы использовали команду `CREATE TABLE`. Дальше идёт её имя, а в скобках — имя, тип и параметры каждого столбца*

*В таблице `courses` для столбца `ID` определён тип `integer` — это значит, что в нём могут храниться только числа. А `varchar` для столбца `Название` говорит о том, что он хранит строку символов переменной длины, но не более 200. Параметр `UNIQUE` указывает, что значения в этом столбце должны быть уникальными, а `NOT NULL` — что они не могут быть `NULL`, то есть отсутствовать*

*В таблице `students` столбец `ID` — первичный ключ `PRIMARY KEY`. Он накладывает ограничения на столбец: значения должны быть `NOT NULL` и уникальными — `UNIQUE`. То есть ограничения равносильны набору параметров `NOT NULL UNIQUE`*

*Для первичного ключа всегда создается индекс - он может быть только один для таблиц* 

*Столбец `ФИО` создан с типом переменной строки не более 100 символов, при этом значение обязательно должно быть `NOT NULL`. Это достигается проверкой `CHECK (ФИО <> '')`. То есть при создании или изменении записей в таблице `students` для этого столбца будут проверяться эти условия. Так мы гарантируем, что в нём не будет пустых значений*

*Столбец `Курс` — числовой, и по нашей задумке должен хранить ID курса из таблицы `courses`*

***Первичный ключ можно создавать на основе нескольких столбцов:*** 
- `CREATE TABLE test (ID integer, Название varchar(200), PRIMARY KEY (ID, Название) );`

***Один и тот же результат можно получить разными путями. К примеру, можно создать пустую таблицу, а столбцы добавить в неё позже через команду изменения таблицы `ALTER TABLE`:***

```
CREATE TABLE test();
ALTER TABLE test 
    ADD COLUMN id integer primary key,
    ADD COLUMN Название varchar(200); 
```

*Команда `ADD COLUMN` и добовляет столбцы*

*Команда записана в несколько строк - часто это удобно, так как запросы могут быть громоздкими* 

`ALTER` - команда изменения чего либо 
`ADD` - добавление 
`DROP` - удаление 

***Изменить тип столбца `Название` и удалить столбец `ID` можно так:*** 

```
ALTER TABLE test 
	ALTER COLUMN Название TYPE varchar(200),
	DROP COLUMN ID; 
```

*У SQL регистронезависимый синтаксис. Это значит, что `Alter = ALTER = AltEr` и так далее. Но если значения в кавычках, регистр будет учитываться, и тогда `id = (не равно) ID`*   

*Если таблица больше не нужна, её можно удалить командой `DROP TABLE`, передав ей имя таблицы. Например, так: `DROP TABLE test;`*

***Кроме первичных ключей есть еще и внешние ключи (foreign key), они описывают связи между записями. Добавим в нашу таблицу студентов связь с таблицей курсов:*** 

`ALTER TABLE students ADD CONSTRAINT course_fk FOREIGN KEY (Курс) REFERENCES courses(id);`

- `ADD CONSTRAINT` - добавляет ограничения к таблице или ее столбцу, то есть условия, которым должны подчиняться данные 
- `course_fk` - имя ограничения 
- `FOREIGN KEY` - добавляет внешний ключ, дальше указывается столбец в скобках, а `REFERENCES courses(id)` ссылается на таблицу `courses` и столбец `id`

***Итого команда означает:*** добавить в таблицу `students` ограничение `course_fk` - внешний ключ, связывающий столбец `Курс` таблицы `students` со столбцом `id` таблицы `courses`

***Нам нужно, чтобы связываемый столбец, указанный в `REFERENCES`, хранил уникальные данные. Добавим это ограничение в нашу таблицу `courses`:***

```
ALTER TABLE courses ADD UNIQUE(id); 
или 
ALTER TABLE courses ADD PRIMARY KEY (id);
```

***Внешний ключ может указывать не только на другую таблицу, но и на свою. Используя это обстоятельство, можно описать древовидную структуру вроде папок в файловой системе:***

```
CREATE TABLE directories (
    dir_id integer PRIMARY KEY,
    parent_id integer REFERENCES directories,
    name varchar(100)
); 
```

*Желательно использовать первичный ключ в каждой таблице - так создается индекс, и при запросе связанных через внешний ключ данных выборка будет работать гораздо быстрее* 

***Но можно создать индекс и самостоятельно - для столбцов или группы столбцов:*** 

`CREATE INDEX ФИО-idx ON students(ФИО);` 

*Эта команда создаст индекс с именем `ФИО_idx` для столбца `ФИО` таблицы `students`*

## ***Команды работы с данными (записями)***

`INSERT INTO` - команда для добавления записей в таблицу

Для команды добавления записи нужно указать название таблицы, а после ключевое слово `VALUES` - данные для добавления 

*Начнём с команды добавления записей в таблицу: `INSERT INTO`. У нас есть внешний ключ у таблицы студентов — это ограничение требует, чтобы столбец `Курс` ссылался на существующие записи в таблице `courses`. Поэтому сначала заполняем таблицу курсов:*

```
INSERT INTO courses VALUES (1, 'Системный администратор'); 
INSERT INTO courses VALUES (2, 'Основы работы с базами данных и SQL'); 
INSERT INTO courses VALUES (3, 'Интернет-маркетолог'); 
INSERT INTO students VALUES (1,'Иванов Павел Алексеевич', 3); 
INSERT INTO students VALUES (2,'Антонов Виктор Васильевич', 1);
```

***Важно соблюдать порядок столбцов, указанный при создании таблицы. Если он неизвестен, можно указывать столбцы явно:*** 

`INSERT INTO students (Курс, ФИО, id) VALUES (3,'Иванов Павел Алексеевич', 1);`

***Так можно получить все записи из таблицы студентов:*** 

```
SELECT * FROM students
```

***Бывает, что столбцов в таблице много, и логичнее выбрать только нужные:***  

```
SELECT ФИО, Курс FROM students;
```

***Выборку можно фильтровать, найдем студента курса 3:*** 

```
SELECT ФИО FROM students WHERE Курс = 3;
```

***Возможна фильтрация и по названию курса:*** 

```
SELECT ФИО FROM students WHERE Курс = (SELECT id FROM courses WHERE Название = 'Интернет-маркетолог');
```

***Если структура данных сложная, использование вложенных запросов становится неудобным и громоздким. В таких случаях используют соединения таблиц: это делает оператор `JOIN`:*** 

```
SELECT ФИО, Название FROM students JOIN courses ON students.Курс = courses.id; 
```

*Эта запись означает, что мы делаем выборку из соединения таблиц `students` и `courses`, и во `FROM` передаётся результат этого объединения.*

***После ключевого слова `ON` идет условие, по которому свяжутся данные из двух таблиц:*** 

- для каждой записи из таблицы `students` будет браться значение поля `Курс`;
- дальше выполнится поиск этого значения в столбце `ID` таблицы `courses`;
- если оно будет найдено, запись из таблицы `students` соединится с записью из таблицы `courses`, и результатом объединения будут данные двух записей;
- если запись не обнаружится, результатом объединения станут только данные записи из таблицы `students`.

*Получается, что к записям в таблице студентов мы присоединяем записи из таблицы курсов, и поэтому число записей в результате соединения будет равно числу записей в таблице `students`*

***При выборке из соединения тоже можно использовать фильтрацию. Найдём студентов с курса «Системный администратор»:***

```
SELECT ФИО, Название FROM students JOIN courses ON students.Курс = courses.id WHERE courses.Название = 'Системный администратор';
```

*Есть ещё одна важная особенность команды `SELECT`: результат одной выборки можно использовать в другой. Для этого нужна временная таблица и предложение `WITH` команды `SELECT`*

```
WITH temp_table AS ( SELECT ФИО, Название FROM students JOIN courses ON students.Курс = courses.id ) SELECT ФИО FROM temp_table WHERE Название = 'Системный администратор';
```

`UPDATE` - команда для изменения самих данных 

***Поменяем курс Павлу Иванову на «Основы работы с базами данных и SQL»:***

```
UPDATE students SET Курс = 2 WHERE ФИО = 'Иванов Павел Алексеевич';
```

***Удалить запись можно так:***

```
DELETE FROM courses WHERE id = 3;
```
### ***Подведём итоги***

Для работы с объектами БД используются команды: `CREATE, ALTER, DROP`
Для работы с записями: `INSERT, UPDATE, SELECT, DELETE`
Для повторного использования результатов запросов: `JOIN, WITH`

***Основной запрос SQL можно описать так:*** 
- Что делаем ?
- Где делаем ?
- Условия выборки ?

***Так же язык позволяет производить вычисления:*** 

```
SELECT count(id) FROM students WHERE Курс=1;
```
