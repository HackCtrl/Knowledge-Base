***База данных*** - это система, предназначенная для хранения и обработки больших массивов структурированных данных 

***Как можно оптимизировать подход к хранению информации. Создать два файла - в одном список студентов, в другом список курсов:*** 
- ***Студенты:*** порядковый номер студента, ФИО студента 
- ***Курсы:*** порядковый номер курса, название курса 

Так мы выделили две сущности - **студенты и курсы**. Теперь нужно добавить связи между ними: 
- ***Студенты:*** порядковый номер студента, ФИО студента, порядковый номер студента 
- ***Курсы:*** порядковый номер курса, название курса 

*Усложняя структуру данных, мы усложняем логику связей между сущностями. Соответственно, становится более трудозатратой и поддержка всего этого* 

***Какие задачи решает система управления базами данных:*** 
- Управление связями 
- Поддержка и целостность информации и описание ее типов 
- Оптимальное хранение 
- Обработка для ускорения доступа 

***База данных*** - это файл с информацией 
***СУБД*** - это приложение, которое умеет работать с такими данными 

***Реляционные БД*** - по другому их называют SQL-базами - информация хранится в виде набора отношений. За счет связей описывается логическая структура данных и обеспечивается их целостность, сама связь - объект к БД, ее неотъемлемая часть 

***Нереляционные БД*** - все остальные БД, например, объектно- и документо-ориентированные, временных рядов, иерархические, сетевые, многомерные и многие другие

***Как понять какую БД использовать:*** 
- ***Реляционная БД*** - это прежде всего про целостность данных 
- ***Нереляционные БД*** - про большую масштабируемость (например, если нужно хранить базу в облаке) и гибкость (если вы, к примеру, много прототипируем, заранее не представляя структуру данных и часто меняя ее)

*Сущность описывает параметры объекта, а сам способ разбиения на сущности зависит от предметной области и решаемой задачи*

*Строки в таблицах называются **записями (row)**, а столбцы **колонками (column)** или **полями**. Чтобы создать связь, нужно указать, какие столбцы в таблицах содержат одинаковые данные* 

*Для работы с данными в БД применяется декларативный язык SQL (Structured Query Language — «язык структурированных запросов»). Это означает, что результат, который нужно получить, описывается без алгоритма действий. Об извлечении или обработке запрошенного результата заботится СУБД*

*Синтаксис в SQL регистронезависимый, `SELECT` и `select` в нём — одно и то же. Но хорошим тоном считается написание управляющих слов в верхнем регистре, поскольку это облегчает чтение сложных запросов*

***Есть база, где данные о студентах хранятся в таблице `students`. Запрос на получение всех данных о студентах:*** 

```
SELECT * FROM students;
```

`SELECT` - команда выгрузки данных 

```
`*` - означает все
```

`FROM` - указывает откуда делать выборку 

***Буквально это означает*** - выгрузить значение всех полей и таблиц students дял всех записей 

***Чтобы ограничить набор записей, используется слово `WHERE`, после которого указываются условия фильтрации. Например, так мы получим данные о студентах старше 25 лет — при условии, что такие данные хранятся в таблице:***

```
SELECT * FROM students WHERE age > 25;
```

*Этим запросом мы указываем, что нужно выгружать данные, у которых параметр age больше 25* 

***Индекс*** - это объект, структурированный под быстрый поиск. Запросы будут использовать его для поиска вместо того, чтобы сканировать записи в таблице 

*Индексы ускоряют поиск, а вот изменение данных с ними замедляется: индексу нужно перестроение - удаление старого и создание нового экземпляра* 

***Создадим индекс в нашей таблице «Студенты» для поля «Курс», чтобы ускорить обращение к нему:***

```
CREATE INDEX стунденты_курс_idx ON студенты (курс);
```

***Добавим к нашей модели данных оценку после прохождения курса — таблицу «Студенты» дополним полем «Оценка». Теперь представим такой запрос: выгрузить всех студентов, у которых оценка за курс «Системный администратор» 4 и 5:***

```
SELECT * FROM студенты WHERE курс = 1 AND оценка > 3;
```

***У нас уже есть индекс по полю «Курс», но нет для поля «Оценка». И если мы будем часто выполнять такой запрос, есть смысл определить составной индекс:***

```
CREATE INDEX студенты_курс_idx ON студенты (курс, оценка);
```

***Ещё индекс можно создать для выражения. Так, названия курсов у нас могут быть записаны по-разному — «Системный Администратор», «системный администратор» или «Системный администратор». Чтобы гарантированно найти любой вариант, понадобится функция `lower()`:***

```
SELECT * FROM Курсы WHERE lower('Название') = 'системный администратор';
```

***Если у нас будет построенный по этому полю индекс, он не ускорит запрос: СУБД проиндексирует значения, которые хранятся в поле в исходном виде. Для ускорения нужен индекс по выражению:***

```
CREATE INDEX курсы_курс_lower_idx ON Курсы(lower('Название');
```

*По умолчанию команда `CREATE INDEX` создаёт индексы типа B-дерево — в большинстве случаев они эффективны, но есть и другие. От типа индекса зависят алгоритмы обработки операторов сравнения: например, хеш-индекс ускоряет работу только с оператором `=`*
