# ***Application Security*** 

***Application Security Engineer*** - это инженер, который специализируется на безопасности приложений. Он работает совместно с командой разработки и привносит свойства безопасности приложения. Задачей AppSec-специалиста является внедрение поддержка и развитие безопасного цикла разработки ПО (SSDLC)
### ***Роль AppSec в ИБ*** 

- ***Взлом приложений*** - один из наиболее частых векторов проникновения в периметр компании
- Чтобы завладеть чувствительными данными, не обязательно проникнуть внутрь периметра компании, если приложение в интернете работает с этими данными 
- Разработкой современного приложения могут быть заняты десятки и сотни IT-специалистов. Огромный объем кода необходимо делать безопасным 
# ***Безопасная разработка приложений*** 

### ***Развитие SSDLC*** 

- Развивающийся продукт циклично проходит через все стадии SDLC 
- Практик AppSec много, их систематизируют фреймворки BSIMM, OWASP SAMM 
- В компаниях практики развиты не равномерно

### ***Основные практики AppSec***

- Планирование продукта 
- Обучение сотрудников 
- Применение автоматизации (DevSecOps)
- Тестирование безопасности 
- Сопровождение

### ***Планирование продукта***

***Взаимодействие AppSec с командами разработки начинается еще до разработки:*** 
- Проектирование 
- Моделирование угроз 
- Базовые требования к безопасности 
- Ревью архитектуры 
### ***Обучение***

- Подготовка гайдов, мануалов для изучения 
- Проведения занятий с командами разработки 
- Разбор найденных уязвимостей 
- Разработка плагинов для IDE 
### ***DevSecOps*** 

***DevSecOps*** - это методика применения лучших практик безопасности на всех этапах непрерывного цикла создания ПО 

![[Pasted image 20260314112406.png]]

- Разворачивание автоматизированных инструментов и интеграция в CI/CD Pipeline, чтобы помочь специалистам ИБ 
- SAST, DAST, SCA 
- Инструменты ищут уязвимости хуже ручных проверок, но они способны работать на несравнимо больших объемах и давать хорошие результаты 
- Специалисты также занимаются оптимизацией работы инструментов, разбором срабатываний и заведением задач в таск-трекерах 
### ***Тестирование безопасности*** 

***Ручное динамическое тестирование готового экземпляра приложения:*** 
- Разведка 
- Сканирование 
- Анализ уязвимостей 
- Эксплуатация 
- Отчетность 
- Повторное тестирование 

- При правильном построении процесса, большое количество информации о продукте известно заранее 
- В AppSec тестирование выполняется белым ящиком 
- Работы не заканчиваются отчетом об уязвимостях 
- Необходимые инструменты: Burp Suite, nuclei, IDE
### ***Сопровождение*** 

- Мониторинг 
- Непрерывное тестирование, периодические аудиты 
- Тестирование на проникновение 
- Bug Bounty
