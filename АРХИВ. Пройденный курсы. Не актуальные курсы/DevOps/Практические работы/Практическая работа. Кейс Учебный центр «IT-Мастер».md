**Цель:** Закрепить на практике команды SQL для создания и изменения структуры базы данных, а также базовые операции с данными.

**Среда:** PostgreSQL на Windows (используйте pgAdmin или psql в командной строке).

**Описание сценария:** Вы — стажёр, которому поручили создать простую базу данных для отслеживания учебных групп и студентов в небольшом учебном центре.

### **Часть 1: Подготовка рабочего пространства**

1. Открыл инструмент для работы с **PostgreSQL** pgAdmin 
2. Создал новую БД 

```
CREATE DATABASE education_center_db;
```

### **Часть 2: Создание структуры данных (таблиц)**

В учебном центре есть **учебные группы** и **студенты**, которые на них записаны. Создадим две связанные таблицы.

Создадим таблицу study_groups (учебные группы). У каждой группы будет уникальный номер, название и преподаватель: 

```
CREATE TABLE study_groups (
    group_id integer,
    group_name varchar(150) NOT NULL UNIQUE,
    teacher_name varchar(100)
);
```

Создадим таблицу students (студенты). У каждого студента будет номер, ФИО и ссылка на группу, в которой он учится: 

```
CREATE TABLE students (
    student_id integer PRIMARY KEY,
    full_name varchar(120) NOT NULL CHECK (full_name <> ''),
    group_number integer
);
```

### **Часть 3: Изменение структуры и создание связей**

Исправим таблицу stude_groups. Столбец group_id должен быть первичным ключом, чтобы на него можно было ссылаться: 

```
ALTER TABLE study_groups ADD PRIMARY KEY (group_id);
```

Дальше создадим связь между таблицами. Столбец group_number в таблице students должен ссылаться на существующие группы. Это внешний ключ FOREIGH KEY

```
ALTER TABLE students 
ADD CONSTRAINT group_link_fk
FOREIGN KEY (group_number)
REFERENCES study_groups(group_id);
```

Теперь БД не позволит записать студента в несуществующую группу

Создадим индекс для ускорения поиска студентов по группе. Хоть внешний ключ и создаст индекс, сделаем это явно для практики: 

```
CREATE INDEX students_group_idx ON students (group_number);
```

### **Часть 4: Работа с данными (записями)**

Добавим учебные группы. Так как у студентов есть внешний ключ на  группы, сначала нужно создать группы: 

```
INSERT INTO study_groups VALUES
(101, 'Веб-разработка с нуля', 'Анна Сергеева'),
(102, 'Администрирование Linux', 'Игорь Петров'),
(103, 'Основы Python', 'Мария Иванова');
```

Добавим студентов. Указываем ID студента, ФИО и номер существующий группы: 

```
INSERT INTO students VALUES
(1, 'Смирнов Алексей Дмитриевич', 101),
(2, 'Ковалева Елена Викторовна', 103),
(3, 'Федоров Павел Игоревич', 102),
(4, 'Никитина Ольга Сергеевна', 101);
```

Посмотрим, что получилось. Выберем все данные из таблицы студентов: 

```
SELECT * FROM students;
```

### **Часть 5: Запросы и выборка данных (SELECT)**

Выберем только имена студентов и номера их групп: 

```
SELECT full_name, group_number FROM students;
```

![[Pasted image 20260210130752.png]]

Найдем всех студентов, которые учатся в группе 101 (веб-разработка): 

```
SELECT full_name FROM students WHERE group_number = 101;
```

![[Pasted image 20260210130934.png]]

Соединим таблицы (JOIN), чтобы увидеть не номера группы, а ее название: 

```
SELECT students.full_name, study_groups.group_name
FROM students
JOIN study_groups ON students.group_number = study_groups.group_id;
```

![[Pasted image 20260210131814.png]]

Объяснение: для каждой записи в `students` найти запись в `study_groups`, где `group_number` студента равен `group_id` группы, и вывести их вместе

Усложним. Найдем всех студентов конкретного курса, используем JOIN и WHERE: 

```
SELECT students.full_name
FROM students
JOIN study_groups ON students.group_number = study_groups.group_id
WHERE study_groups.group_name = 'Администрирование Linux';
```

Делаем то же самое, но с использованием вложенных запросов: 

```
SELECT full_name FROM students
WHERE group_number = (
    SELECT group_id FROM study_groups
    WHERE group_name = 'Администрирование Linux'
);
```

### **Часть 6: Изменение и удаление данных**

Переведем студента в другую группу (UPDATE). Алексей Смирнов перешел с веб-разработки на Python: 

```
UPDATE students
SET group_number = 103
WHERE full_name = 'Смирнов Алексей Дмитриевич';
```

![[Pasted image 20260210132753.png]]

Удалим учебную группу (DELETE). Пробуем удалить группу 'Основы Python' (id=103): 

```
DELETE FROM study_groups WHERE group_id = 103;
```

**Что произошло?** Должна возникнуть **ОШИБКА!** База данных не позволит этого сделать, потому что на эту группу ссылаются студенты (Ковалева и теперь Смирнов). **Внешний ключ защищает целостность данных.**

Сделаем правильное удаление. Сначала нужно удалить либо студентов из этой группы, либо перевести их в другие группы. Удалим запись о студенте Ковалевой: 

```
DELETE FROM students WHERE full_name = 'Ковалева Елена Викторовна';
```

Теперь в группе 103 остался один студент. Чтобы удалить саму группу, нужно либо удалить и его, либо изменить его `group_number` на `NULL` (если бы столбец это позволял). Для простоты удалим и его: 

```
DELETE FROM students WHERE full_name = 'Смирнов Алексей Дмитриевич';
-- Теперь группа 103 пуста. Можно ее удалить.
DELETE FROM study_groups WHERE group_id = 103;
```
