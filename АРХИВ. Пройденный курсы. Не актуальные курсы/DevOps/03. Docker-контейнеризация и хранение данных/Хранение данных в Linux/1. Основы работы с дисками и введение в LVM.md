***Том (volume, дословно — «объём» или «размер»)*** - логическая единица дискового пространства 

***LVM** - новый метод управления логическими томами данных. Представлял абстракцию над физическими дисками и позволял объединять их в группы, а затем создавать логические тома с динамическими изменением размеров и другими расширенными функциями*

***LVM*** ***(Logical Volume Manager)*** - это технология управления хранилищем данных в ОС Linux. Она предоставляет абстракцию над физическими устройствами хранения и позволяет создавать логические тома данных, которые могут охватывать несколько физических дисков или разделов

***Основные достоинства LVM:*** 
1. Гибкость в управлении дисками 
2. Управление пространством хранения 
3. Снимки (snapshot) и резервное копирование 
4. Удобное перемещение данных 
5. Удобное объединение и разделение томов 
6. Высокая доступность данных 
7. Горячее добавление и удаление дисков 

***LVM управляет тремя основными сущностями:*** 

1. ***Физический том (Physical Volume, или PV)*** - раздел на физическом диске или весь диск целиком 
2. ***Группа физических томов (Volume Group, или VG)*** - объединение нескольких физических томов под одним логическим именем 
3. ***Логический том (Logical Volume, или LV***) - логический раздел, который можно отформатировать и перемонтировать как обычный раздел физического диска 

***Как все работает:*** 
1. Сначала под управление LVM нужно отдать физический диск или его раздел и для этого инициализировать его как физический том (PV).
2. Затем из PV создаётся группа физических томов (VG).
3. В VG создаются логические тома (LV), которые можно отформатировать с помощью конкретной файловой системы и использовать для хранения данных.

***Полезные команды и утилиты для работы с дисками в Linux:*** 

`lsblk` - можно увидеть добавленный диск и общий список дисков на машине 
`-f (флаг)` - будут выведены еще и ФС на дисках

## ***Итоги***

***LVM*** — довольно гибкий инструмент для сопровождения дисковой подсистемы в инфраструктуре. Помимо увеличения и уменьшения размера логических томов он позволяет делать снимки для резервного копирования, перемещать разделы между физическими дисками, создать зеркало из двух дисков, где один дублирует другой

| `NAME`       | Имя диска                                                                                    |
| ------------ | -------------------------------------------------------------------------------------------- |
| `FSTYPE`     | Тип файловой системы.                                                                        |
| `LABEL`      | Метка. Может содержать человекочитаемое описание диска.                                      |
| `UUID`       | Уникальный идентификатор файловой системы.                                                   |
| `FSAVAIL`    | Место на диске, доступное для файловой системы. Обычно оно меньше физического размера диска. |
| `FSUSE`      | Сколько места на диске занято.                                                               |
| `MOUNTPOINT` | Точка монтирования.                                                                          |
***Есть два типа таблиц разметки - MBR (для устройств с BIOS) и GPT (для устройств с UEFI)***

- ***MBR (Master Boot Record)*** - традиционная структура разметки дисков. Содержит два типа разделов: первичный (primary partition) и расширенный (extended partition). Всего может быть четыре первичных раздела. Если нужно создать больше логических дисков, дело за расширенным разделом — его можно разбить на любое их количество
- ***GPT (GUID Partition Table)*** - новый стандарт разметки и часть стандарта UEFI. Допускает неограниченное количество разделов, но есть ограничения со стороны ОС: к примеру, Windows позволяет создавать не более 128
