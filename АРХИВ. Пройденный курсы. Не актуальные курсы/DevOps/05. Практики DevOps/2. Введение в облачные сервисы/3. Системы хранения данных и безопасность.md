## ***Работа с блочными устройствами***

***Процесс создания диска:*** 
1. В консоли управления выбираем каталог, в котором нужно создать диск, а затем — сервис Compute Cloud. На панели слева: «Диски» → «Создать диск».
2. Введём имя диска, но учтите, что требования к нему такие: длина — от 3 до 63 символов, он может содержать строчные буквы латинского алфавита, цифры и дефисы, при этом первым символом должна быть буква, а последним — не дефис.
3. Следующий шаг опциональный: добавляем произвольное описание диска. Зону доступности оставьте по умолчанию.
4. В поле «Тип» есть несколько опций: HDD, SSD и SSD IO, или нереплицируемый SSD. Именно это ключевой параметр при выборе диска.

***Максимальное значение IOPS*** - это максимальное количество операций чтения и записи, выполняемых диском в секунду. В МБ/с измеряется максимальная пропускная способность (bandwidth) - суммарное количество байт, которое можно прочитать с диска или записать на диск в секунду

Теперь немного о SSD IO — SSD с повышенной производительностью. Этот вариант превосходит по производительности сетевые SSD, но при этом размер дисков должен быть кратен 93 ГБ. Во всех расчётах 1 ГБ = 230230 байт

- **сетевой SSD-диск** (network-ssd) — быстрый сетевой диск, сетевое блочное хранилище на SSD-накопителе;
- **сетевой HDD-диск** (network-hdd) — стандартный сетевой диск, сетевое блочное хранилище на HDD-накопителе;
- **нереплицируемый SSD-диск** (network-ssd-nonreplicated) — сетевой диск с повышенной производительностью без избыточности;
- **высокопроизводительный SSD-диск** (network-ssd-io-m3) — обладает теми же скоростными характеристиками, что и network-ssd-nonreplicated, и одновременно обеспечивает избыточность.
## ***Создание снимков***

***Это копия ФС диска на конкретный момент. Снимка нужны для:*** 
- Переноса данных с одного диска на другой 
- Создание резервной копии диска перед операциями, которые могут повлечь за собой повреждение данных 
- Версионирование диска через регулярное создание снимков 

*Снимок содержит только те данные, которые были записаны в момент его создания. Если диск подключен к работающей виртуальной машине, то кеш приложений и ОС не пападет снимок* 

***Образ*** - это полная копия структуры ФС и данных, находящихся на диске 
## ***Работа с объектными хранилищами***

***Бакет*** - это логический контейнер для организации и хранения данных в облачных системах 
## ***Используем бакеты***

***GeeseFS*** — программа для монтирования бакетов в системах Linux, macOS и Windows

***Пример настройки на ВМ и сначала подключимся к ней по по `ssh`. Сперва установим утилиты для работы:***

```
sudo apt-get install fuse 
```

Затем загрузим и установим GeeseFS:

```
wget <https://github.com/yandex-cloud/geesefs/releases/latest/download/geesefs-linux-amd64> 
```

```
chmod a+x geesefs-linux-amd64 
```

```
sudo cp geesefs-linux-amd64 /usr/bin/geesefs 
```

GeeseFS использует статический ключ доступа к Object Storage. Он задаётся несколькими способами, вы можете использовать любой:

1. С помощью файла credentials, который нужно поместить в директорию `~/.aws/`:

```
[default]
  aws_access_key_id=<идентификатор_ключа>
  aws_secret_access_key=<секретный_ключ>
 
```

2. С помощью переменных окружения:

```
export AWS_ACCESS_KEY_ID=<идентификатор_ключа>
export AWS_SECRET_ACCESS_KEY=<секретный_ключ> 
```

Теперь примонтируем бакет командой:

```
sudo geesefs <имя_бакета> <точка_монтирования> 
```

В нашем случае она выглядит так:

```
sudo geesefs <http://storage.yandexcloud.net/test-bucket-sysadmin> /media/ 
```
## ***Безопасность при работе в облаке***

***Группа безопасности*** - это набор правил для исходящего и входящего трафика, который можно назначить на сетевой интерфейс ВМ 

***Группы работают как stateful firewall:*** отслеживают состояние сессий, и если правило разрешает сессию, ответный трафик также автоматически разрешается.

***Key Management Service (KMS)*** - сервис для шифрования данных и управления ключами шифрования
