- `ipconfig` - команда для проверки сетевых интерфейсов в ОС Windows 
- `ifconfig` - команда для проверки сетевых интерфейсов в Linux 

***NAT (Network Address Translation — преобразование сетевых адресов)*** - это технология в сетях TCP/IP, которая позволяет преобразовывать IP-адреса транзитных пакетов данных при их прохождении через маршрутизатор или брандмауэр. Она позволяет нескольким устройствам в локальной сети использовать один публичный IP-адрес для доступа в интернет, а так же скрывать внутренний IP-адрес устройств от внешнего мира 

***Транзитные пакеты*** - это пакеты данных, которые проходят через сетевое устройство (маршрутизатор, коммутатор), но не предназначены для самого этого устройства. Проще говоря: пакет летит от отправителя к получателю, а это устройство лишь «транзитом» переносит его дальше по сети
### ***Глобальные, частные и зарезервированные адреса*** 

***Глобальные (белые)*** - это те адреса, которые глобально маршрутизируются в среде интернет
- ***Пример:*** 
	- 64.233.162.138, 77.88.55.55, 81.176.66.163

***Приватные (серые)*** - такие адреса были предназначены для использования в локальных сетях, которые не подключены к сети интернет 
- ***Пример:*** 
	- 192.168.1.1, 10.0.0.1, 100.100.46.203

***Localhost*** - частные IP-адреса, часто используются в домашних сетях, в сетях предприятий 
- ***Пример:*** 
	- 172.0.0.1, 127.0.1.1
	- Сеть класса А: 10.0.0.0/255.0.0.0
	- 16 сетей класса B: 172.16.0.0/255.0.0.0, 172.31.0.0/255.0.0.0
	- 256 сетей класса C: 192.168.0.0/255.255.255.0, 192.168.255.0/255.255.255.0
	- Для провайдерских высоконагруженных NAT: 100.64.0.0/255.255.0.0, 100.128.0.0/255.255.0.0
	- Для авто получения адреса без DHCP: технология Zeroconf - Zero Configuration Network она же APIPA - Automatic Private IP Addressing: 192.254.0.0/255.255.0.0

***Zeroconf (Zero Configuration Networking)*** - это набор технологий, который позволяет устройствам автоматически создавать работающую IP-сеть без ручной настройки и без специальных серверов (DHCP, DNS)

***APIPA (Automatic Private IP Addressing)*** - это конкретный механизм, который работает внутри Zeroconf. Его часто упоминают, когда говорят о том, как устройства в сети временно решают проблему, если не смогли получить адрес от DHCP сервера

### ***Мультикастные группы*** 

***Например:*** 
- (Адреса класса D) - 224.1.1.1, 230.2.3.4

***Мультикастные группы*** - это способ отправить данные от одного источника сразу нескольким заинтересованным получателям одновременно, без необходимости создавать отдельную копию потока для каждого из них

***Адреса класса E***
