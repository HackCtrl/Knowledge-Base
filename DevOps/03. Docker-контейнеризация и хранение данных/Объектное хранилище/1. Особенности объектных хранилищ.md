## ***Объектные хранилища***

***Объектное хранилище*** - это способ хранения, где у каждого файла, или объекта, есть уникальная идентификация и метаданные. Вместо традиционных папок и подпапок объекты хранятся независимо друг от друга 

***Идентификатор объекта*** - представляет собой строку. Ее часто записывают как путь в ФС с символом `/` в качестве разделителя - например: `my_project/banner.png`

*Объектное хранилище можно представить в виде большой кладовки, куда мы складываем вещи (объекты). Чтобы логически сгруппировать их, мы распределим их по коробкам -* **"бакетам"**
#### ***Особенности объектных хранилищ***

*Они предназначены для файлов, которые не нужно менять. Чтобы изменить объект, придется загрузить в хранилище его новую версию.* 
*Основной способ взаимодействия с хранилищем - по HTTP API, поэтому его широко используют веб-приложения*

## ***Примеры использования***

Об объектном хранилище стоит задуматься, когда речь идёт о взаимодействии процессов по HTTP-протоколу. Доставка статического контента — как раз такой случай.

*В таком случае логика веб-сайта размещается на отдельном сервере, а всё остальное — например изображения, — на объектном хранилище. Так, если ваше приложение реализует функции видеохостинга, сами видеофайлы можно разместить на объектном хранилище.*

Такое хранилище подойдёт также в качестве репозитория пакетов или кода. Оно обеспечит гибкий доступ к файлам из любой точки мира, и разработчики смогут работать над проектами удалённо и совместно.

Ещё такую систему можно использовать для обычного хранения файлов. Например, если веб-сервис тесно взаимодействует с некоторой базой данных, разместить саму БД в объектном хранилище не получится. А вот хранить бэкапы этой базы — вполне!
### ***Подведём итоги***

- Объектное хранилище — система хранения данных произвольного формата, подходящая для видеозаписей, изображений, архивов, резервных копий, пакетов и кода. Она обеспечивает их доступность, защиту от несанкционированного доступа и масштабируемость.
- Идентификатор объекта — строка, идентифицирующая объекты в хранилище, а метаданные содержат дополнительные сведения о них.
- Объекты в хранилище группируются в бакеты, что позволяет разделять данные разных проектов или пользователей.
- Среди особенностей объектных хранилищ — взаимодействие с приложениями через HTTP API с использованием HTTP-методов, невозможность редактирования объектов
