Представим, что у нас есть онлайн-школа, которая ведёт подготовку по четырём направлениям: «Квантовый кодинг», «Астроинформатика», «Технологии долголетия» и «Технологии устойчивого развития». Для каждого факультета — свой проходной балл.

Десять студентов уже сдали проверочные тестирования и получили свои результаты — от 0 до 10 баллов. Занесём эти данные в нашу БД, создав две коллекции:

1. Студенты: порядковый номер, имя, фамилия, возраст, оценка.
2. Факультеты: порядковый номер, название, проходной балл.

***Создадим БД:*** 
```
use yp_db
```

![[Pasted image 20260223124639.png]]

***Теперь создадим в этой базе две коллекции - `students` и `faculties`:*** 
```
db.createCollection('students', {validator: {$jsonSchema: {bsonType: 'object',required: ['ID', 'firstName', 'lastName', 'age', 'gpa'],properties: {ID: { bsonType: 'number' },firstName: { bsonType: 'string' },lastName: { bsonType: 'string' },age: { bsonType: 'number' },gpa: { bsonType: 'number' }}}}}); db.createCollection('faculties', {validator: {$jsonSchema: {bsonType: 'object',required: ['ID', 'Faculty', 'gpa'],properties: {ID: { bsonType: 'number' },Faculty: { bsonType: 'string' },gpa: { bsonType: 'number' }}}}});
```

***Валидационная схема JSON*** - это набор правил, которые определяют структуру и содержимое документов в коллекции MongoDB. Такие схемы гарантируют соответствие данных определенным стандартам, что делает их более надежными и управляемыми 

***Валидационные схемы JSON определяются в объекте `$jsonSchema` и могут включать такие свойства:***
- `bsonType` определяет тип документа или поля. Например, это может быть объект, строка или число 
- `required` список обязательных полей 
- `properties` задает типы данных и ограничения для вложенных полей 
- `additionalProperties` указывает, разрешены ли дополнительные свойства, не заданные в схеме 
- `pattern` регулярные выражения, которому должны соответствовать значения для данного поля 

***Регулярное выражение*** - это шаблон для описания набора строк. Он представляет собой последовательность символов, определяющих условие соответствия. Для сопоставления строка должна удовлетворять им 

- `minimum` и `maximum` задают минимальное и максимальное значение для числовых полей 
- `minLength` и `maxLength` указывает минимальную и максимальную длину для строковых полей 

***Части строк кода для коллекции `students`:*** 
- `db.createCollection('students')` создает новую коллекцию с именем students в текущей БД 
- `{validator: {$jsonSchema: ...}}` определяет валидационную схему JSON для коллекции students
- `{bsonType: 'object', required: ['ID', 'firstName', 'lastName', 'age', 'gpa'], properties: {...}}` сама валидационная схема JSON. Что она определяет: 
	- `bsonType: 'object'` указывает, что документ должен быть объектом 
	- `required: ['ID', 'firstName', 'lastName', 'age', 'gpa']` указывает, что поля ID, firstName, lastName, age, gpa обязательны для каждого документа в коллекции 
	- `properties` определяет типы данных для каждого из обязательных полей 

- ID: должно быть числом (bsonType: 'number')
- firstName: должна быть строкой (bsonType: 'string')
- lastName: должно быть строкой (bsonType: 'string')
- age: должно быть числом (bsonType: 'number')
- gpa: должна быть числом (bsonType: 'number')

*Таким образом, эта строка кода создаёт коллекцию `students` с валидационной схемой, которая требует, чтобы у всех документов были обязательные поля `ID`, `firstName`, `lastName`, `age` и `gpa` с соответствующими типами данных, а также определённая структура.*

***Значение `_id` или идентификатор документа*** - это специальное поле, которое автоматически создается и добавляется в каждый документ, вставленный в коллекцию MpngoDB. Так каждый документ в коллекции однозначно идентифицируется 

***Рассмотрим характеристики `_id`:***
- Уникален для каждого документа в коллекции 
- Не может быть изменён или удален из коллекции 
- С типом данных `ObjectId`. `ObjectId` - это 12 байтовое значение, которое содержит: 
	- 4 байта для инкрементного счетчика 
	- 3 байта для отметки времени 
	- 5 байт для случайного компонента 
- Создается автоматически при вставки документа, если не указан явно 

*При вставке документа мы можем указать собственное значение id, но оно должно быть уникальным* 

***У `_id` важная роль в MongoDB. Он используется:***
- В качестве первичного ключа для коллекции 
- Для индексации документов, что ускоряет запросы 
- Для связывания документов между коллекциями с использованием внешних ключей 

***Но вывод запроса `db.students.find()` недостаточно информативен, поэтому немного усовершенствуем запрос:***
```
db.students.find({}, { _id: 0, firstName: 1, lastName: 1, gpa: 1 })
```

![[Pasted image 20260223134640.png]]

`find()` - команда для документов в коллекции. Первый аргумент в ней - фильтр запроса, у нас он пустой (`{}`), это означает, что будут возращены все документы. Второй аргумент - критерии отбора данных 

`{ _id: 0, firstName: 1, lastName: 1, gpa: 1 }` - критерии отбора данных. Они определяют, какие поля включить в результирующие документы. В нашем случае это `_id: 0`, исключающий поле `_id` из результатов 

`_id` - единственное поле, которое нужно явно исключить из проекции. Для исключения других полей достаточно просто не включать их в поиск 

*`firstName: 1` включает поле `firstName`, аналогично с `lastName: 1` и `gpa: 1`*

*Чтобы отсортировать коллекцию по высшему баллу в порядке возрастания — от 0 до 10, — можно дополнить запрос конструкцией `.sort({ gpa: 1 })`*

`.sort({ gpa: -1 })` будет сортировать в обратном порядке. При сортировке по имени 1 будет означать порядок от A до Z, а -1 - наоборот 

***Итоговый запрос будет выглядеть так:*** 
```
db.students.find({}, { _id: 0, ID: 1, firstName: 1, lastName: 1, gpa: 1 }).pretty({ full: true, shell: true, query: false }).sort({ gpa: 1 })
```

***Индекс в MongoDB*** - это структура данных, которая ускоряет выполнение запросов, особенно фильтрующих или сортирующих на основе определённых полей. Индексы организуют данные так, чтобы БД могла быстро найти нужные документы без сканирования всей коллекции 

*MongoDB хранит индексы в отдельной структуре данных, называемой B-деревом.*

***B-дерево*** - это сбалансированное дерево, которое позволяет быстро искать данные по упорядоченному ключу 

Индексы улучшают производительность сортировки, но их создание и поддержка требует дополнительных ресурсов. Поэтому важно тщательно продумывать, какие индексы создавать

***Создадим индекс по полю `firstName` в коллекции `students`:***
```
db.students.createIndex({ firstName: 1 })
```

## ***Изменение и внесение данных в БД***

`update` - в MongoDB обновляет существующие документы в коллекции 

***Синтаксис:*** 
```
db.collection_name.update({ filter }, { update })
```

***Параметры:*** 
- `filter` - фильтр соответствующий документам, которые нужно обновить 
- `update` - оператор обновления. Он определяет изменения, которые нужно внести в соответствующие документы 

***Операторы обновления:*** 
- `$set` - заменяет существующие значения полей указанными 
- `$unset` - удаляет указанные поля из документа 
- `$push` - добавляет значения в массив 
- `$pull` - удаляет значение из массива 
- `$inc` увеличивает значение числового поля на указанное значение 
- `$mul` - умножает значение числового поля на указанное значение 

***Вернёмся к нашей БД. Если внимательно рассмотреть выданную по коллекции `students` информацию, можно заметить, что некоторые имена и фамилии написаны кириллицей. Обновим данные, переписав кириллицу в латиницу:***
```
db.students.updateOne({ ID: 6 }, { $set: { firstName: 'Emel', lastName: 'Emelyanova' } }) db.students.updateOne({ ID: 7 }, { $set: { firstName: 'Robert', lastName: 'Robertov' } })
```

![[Pasted image 20260223140531.png]]

***Разберём первую строку:***
- `db.students` обращается к коллекции students в текущей БД 
- `updateOne()` обновляет один документ, соответствующий указанному фильтру 
- `({ ID: 6 })` фильтр, который соответствует документу со значением поля  ID: 6
- `({ $set: { firstName: 'Emel', lastName: 'Emelyanova' } })` оператор обновления, который устанавливает значения полей firstName и lastName

#### ***Операции удаления: `delete` и `drop`***

`delete` - используется для удаления одного или нескольких документов из коллекции по определенному критерию 

***Синтаксис:*** 
```
db.collection.delete(<query>, <options>)
```

`<query>` - критерий выборки документов, которые нужно удалить 
`<options>` - опциональные параметры вроде limit, которые определяют максимальное количество документов для удаления 

***Этот код удалит один документ из коллекции `users`, где поле `name` равно `"Anna"`:***
```
db.users.deleteOne({ name: "Anna" })
```

![[Pasted image 20260223141302.png]]

`drop` - используется для удаления всей коллекции 

***Синтаксис:*** 
```
db.collection.drop()
```

***Чтобы удалить БД целиком, в MongoDB Shell используется команда `dropDatabase()`. Так, например, мы удалим `mydatabase`:***
```
use mydatabase db.dropDatabase()
```
