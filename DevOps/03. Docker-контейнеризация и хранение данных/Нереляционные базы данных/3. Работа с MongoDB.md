***Валидационная схема JSON*** - это набор правил, которые определяют структуру и содержимое документов в коллекции MongoDB. Такие схемы гарантируют соответствие данных определенным стандартам, что делает их более надежными и управляемыми 

***Валидационные схемы JSON определяются в объекте `$jsonSchema` и могут включать такие свойства:***
- `bsonType` определяет тип документа или поля. Например, это может быть объект, строка или число 
- `required` список обязательных полей 
- `properties` задает типы данных и ограничения для вложенных полей 
- `additionalProperties` указывает, разрешены ли дополнительные свойства, не заданные в схеме 
- `pattern` регулярные выражения, которому должны соответствовать значения для данного поля 

***Регулярное выражение*** - это шаблон для описания набора строк. Он представляет собой последовательность символов, определяющих условие соответствия. Для сопоставления строка должна удовлетворять им 

- `minimum` и `maximum` задают минимальное и максимальное значение для числовых полей 
- `minLength` и `maxLength` указывает минимальную и максимальную длину для строковых полей 

***Части строк кода для коллекции `students`:*** 
- `db.createCollection('students')` создает новую коллекцию с именем students в текущей БД 
- `{validator: {$jsonSchema: ...}}` определяет валидационную схему JSON для коллекции students
- `{bsonType: 'object', required: ['ID', 'firstName', 'lastName', 'age', 'gpa'], properties: {...}}` сама валидационная схема JSON. Что она определяет: 
	- `bsonType: 'object'` указывает, что документ должен быть объектом 
	- `required: ['ID', 'firstName', 'lastName', 'age', 'gpa']` указывает, что поля ID, firstName, lastName, age, gpa обязательны для каждого документа в коллекции 
	- `properties` определяет типы данных для каждого из обязательных полей 

- ID: должно быть числом (bsonType: 'number')
- firstName: должна быть строкой (bsonType: 'string')
- lastName: должно быть строкой (bsonType: 'string')
- age: должно быть числом (bsonType: 'number')
- gpa: должна быть числом (bsonType: 'number')

*Таким образом, эта строка кода создаёт коллекцию `students` с валидационной схемой, которая требует, чтобы у всех документов были обязательные поля `ID`, `firstName`, `lastName`, `age` и `gpa` с соответствующими типами данных, а также определённая структура.*

***Значение `_id` или идентификатор документа*** - это специальное поле, которое автоматически создается и добавляется в каждый документ, вставленный в коллекцию MpngoDB. Так каждый документ в коллекции однозначно идентифицируется 

***Рассмотрим характеристики `_id`:***
- Уникален для каждого документа в коллекции 
- Не может быть изменён или удален из коллекции 
- С типом данных `ObjectId`. `ObjectId` - это 12 байтовое значение, которое содержит: 
	- 4 байта для инкрементного счетчика 
	- 3 байта для отметки времени 
	- 5 байт для случайного компонента 
- Создается автоматически при вставки документа, если не указан явно 

*При вставке документа мы можем указать собственное значение id, но оно должно быть уникальным* 

***У `_id` важная роль в MongoDB. Он используется:***
- В качестве первичного ключа для коллекции 
- Для индексации документов, что ускоряет запросы 
- Для связывания документов между коллекциями с использованием внешних ключей

## ***Агрегационные пайплайны***

***Агрегационный пайплайн*** - это механизм для выполнения операций агрегации данных, например суммирование, сортировка, группировки и других, в коллекциях MongoDB 

***С помощью агрегационного пайплайна в MongoDB может выполнять такие операции:*** 
- Фильтровать 
- Группировать 
- Преобразовывать 
- Сортировать 
- Уникализировать 
- Вычислять агрегатные функции, например среднее значение, сумму, минимум, максимум и так далее 

***Основные операторы:***  
1. `$match`фильтрует документы по заданному критерию: 
```
db.users.aggregate([ { $match: { age: { $gt: 30 } } } ])
```

2. `$group` группирует документы по определенному полю и применяет агрегационные функции к данным каждой группы: 
```
db.orders.aggregate([ { $group: { _id: "$product_category", total_sales: { $sum: "$price" } } } ])
```

3. `$project` формирует выборку определённых полей из документов, переименовывает или создает новые поля: 
```
db.users.aggregate([ { $project: { name: 1, _id: 0 } } ])
```

4. `$sort` сортирует документы в результате агрегации по заданному полю: 
```
db.products.aggregate([ { $sort: { price: -1 } } ])
```

5. `$facet` позволяет выполнять несколько агрегационных операций параллельно и объединять их результаты: 
```
db.orders.aggregate([ { $facet: { total_sales: [ { $group: { _id: null, total_sales: { $sum: "$price" } } } ], average_sales: [ { $group: { _id: null, average_sales: { $avg: "$price" } } } ] } } ])
```

6. `$unwind` разбивает массив поля документа на документы поменьше, что упрощает их обработку в пайплайне: 
```
db.students.aggregate([ { $unwind: "$hobbies" } ])
```

7. `$lookup` выполняет операции объединения - `join` - с другой стороны коллекцией MongoDB: 
```
db.orders.aggregate([ { $lookup: { from: "products", localField: "product_id", foreignField: "_id", as: "product_details" } } ])
```

8. `$addFields` добавляет в документе новое поле или обновляет существующее: 
```
db.users.aggregate([ { $addFields: { full_name: { $concat: ["$first_name", " ", "$last_name"] } } } ])
```

9. `$redact` применяет правила доступа к данным на уровне документов: 
```
db.users.aggregate([ { $redact: { $cond: { if: { $gte: ["$age", 18] }, then: "$$KEEP", else: "$$PRUNE" } } } ])
```

***Пример №1.*** *Представим, что у нас есть две коллекции: `users` и `posts`. В первой — информация о пользователях, включая их имя, возраст и ID, а во второй — посты, которые пользователи опубликовали, включая сами тексты, ID пользователей, дату публикации и количество лайков.*

Мы хотим выбрать все посты, которые написаны пользователями старше 30 лет и у которых больше cотни лайков. Результат запроса нужно записать в новую коллекцию `popular_posts`.

Таким может быть наш запрос в `mongosh`:
```
db.users.aggregate([
  {
    $lookup: {
      from: "posts",
      localField: "userId",
      foreignField: "userId",
      as: "userPosts"
    }
  },
  {
    $match: {
      $and: [
        { age: { $gt: 30 } },
        { "userPosts.likes": { $gt: 100 } }
      ]
    }
  },
  {
    $out: "popular_posts"
  }
]) 
```

*Мы используем `$lookup`, чтобы объединить коллекции `users` и `posts` по полю `userId`. Затем с помощью `$match` фильтруем пользователей старше 30 лет и посты, у которых больше ста лайков. Наконец, с помощью `$out` результаты агрегации записываются в новую коллекцию `popular_posts`.*

***Пример №2.** У нас есть две коллекции: `authors` и `books`. Первая содержит информацию об авторах книг, включая их имя, страну и год рождения, а вторая — о самих книгах, включая их название, жанр и ID автора.*

Наша задача — соединить данные из обеих коллекций, выбрать книги, написанные авторами старше 40 лет в жанре фэнтези, и создать новую коллекцию `fantasy_books_over_40`.

Вот пример запроса в `mongosh` для неё:

```
db.authors.aggregate([
  {
    $lookup: {
      from: "books",
      localField: "_id",
      foreignField: "authorId",
      as: "author_books"
    }
  },
  {
    $unwind: "$author_books"
  },
  {
    $addFields: {
      author_age: {
        $subtract: [2022, "$age"]
      }
    }
  },
  {
    $facet: {
      fantasy_books: [
        {
          $match: {
            "author_age": { $gt: 40 },
            "author_books.genre": "fantasy"
          }
        }
      ]
    }
  },
  {
    $unwind: "$fantasy_books"
  },
  {
    $out: "fantasy_books_over_40"
  }
])  
```

Здесь мы используем оператор `$lookup`, чтобы объединить данные из коллекций `books` и `authors` на основе совпадения `ID` автора. Дальше применяем оператор `$unwind` для разворачивания массива `author_books`, а также `$addFields`, чтобы вычислить возраст автора.

Затем используем `$facet` для нескольких этапов агрегации. Мы выбираем книги, написанные в жанре фэнтези авторами старше 40 лет, после чего результат записывается в новую коллекцию `fantasy_books_over_40`.

***Пример №3.** Поработаем с нашей БД. У нас есть пять факультетов и десять студентов. Создадим новую коллекцию `Score` и запишем в неё, на какой курс поступили студенты при условии, что основа поступления — конкурсная. Всего же на каждом факультете по два места.*

***Создаём коллекцию `Score`:***
```
db.createCollection("Score");
```

И решаем поставленную задачу:
```
db.students.find().sort({ gpa: -1 }).forEach(student => {
    let availableFaculty = db.faculties.findOne({ studentCount: { $lt: 2 }, gpa: { $lte: student.gpa }, "admittedStudents": { $nin: [student.lastName] } });
    if (availableFaculty) {
        db.faculties.updateOne({ _id: availableFaculty._id }, {
            $inc: { studentCount: 1 },
            $push: { admittedStudents: student.lastName }
        });
        db.Score.insertOne({
            "lastName": student.lastName,
            "gpa": student.gpa,
            "faculty": availableFaculty.Faculty
        });
    } else {
        print(`Студент ${student.lastName} не был зачислен ни на один факультет.`);
    }
});
```
