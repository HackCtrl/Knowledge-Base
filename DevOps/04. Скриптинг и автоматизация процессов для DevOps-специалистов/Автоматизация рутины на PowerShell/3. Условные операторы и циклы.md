## ***Условные операторы***

*Они используются для проверки условий на соответствие* 

***Конструкция `if-else` в Bash для создания условий могут использоваться круглые или квадратные скобки в зависимости от типа проверки:*** 

- Одиночная или двойная круглая скобка `(...) ((...))` - только арифметические операции 
- Одинарная или двойная квадратная скобка `[...] [[...]]` - все кроме арифметических операций 

***Пример из урока Bash:*** 
```
#!/bin/bash
today=$(date +'%d-%m')
if [ $today = 31-12 ]
  then
  echo 'С Новым годом'
    else
    echo 'Ждём ещё'
fi
```

*В этом примере мы видим квадратные скобки для проверки соответствия строк и важный оператор `then`, который буквально означает «ТО»*

***Тот же пример на PowerShell:*** 
```
$today = Get-Date -Format 'dd-mm'
if ( $today = 31-12 ) {
    echo "С Новым годом"
} else {
    echo "Ждём ещё"
}
```

***Различия конструкции PowerShell и Bash:*** 
- В условии используются круглые скобки, а не квадратные. Важно запомнить, что для условий в PowerShell используются только одиночные круглые скобки 
- Отсутствует оператор `then`, и вместо него используются фигурные скобки `{}`

***Операторы и примеры PowerShell:*** 
- `=` или `-eq` - равно 
```
if ($today = "31-12") {
echo "С новым годом!"
}
```

- `!=` или `-ne` - не равно 
```
if ($today != "31-12") {
echo "Ждем еще"
}
```

- `>` или `-gt` - больше чем 
```
if (5>3) {
echo "Верное равенство"
}
```

- `>=` или `-ge` - больше чем или равно 
```
if (5>=3) {
echo "Верное равенство"
}
```

- `<` или `-lt` - меньше чем 
```
if (3<5) {
echo "Верное равенство"
}
```

- `<=` или `-le` - меньше чем или равно 
```
if (3<=5) {
echo "Верное равенство"
}
```

- `-like` - похоже (не поддерживает регулярные выражения, требует указание символа `*` и/или `?`)
 ```
 if ("Practicum -like "Prac*") {
 echo "Значение истино"
 }
 ```

- `-match` - совпадает (поддерживает регулярные выражения, не требует указания символа `*` и/или символа `?`)
```
if ("Practicum" -match "Prac") {
echo "Значение истино"
}
```

- `-contains` - содержит значение (используется для переборки массивов или объектов)
```
if (1,2,3 -contains 2) {
echo "Значение истино"
}
```

- `-and` - логическое И (оба условия должны быть истинным)
```
if (1,2,3) -contains 2 and "Practicum" -match "Prac") {
echo "Значение истино"
}
```

- `-or` - Логическое ИЛИ (одно из условий должно быть истинной)
```
if (1,2,3 -contains 2 -or "Practicum -match "PowerShell") {
echo "Значение истинно"
} 
```

***Старый скрипт:*** 

```
Param (
[int]$first_int,
[int]$second_int,
[string]$file_name,
[string]$path_name,
[array]$arr
)

$ErrorActionPreference='silentlycontinue'

function make_message([int]$value){
   $message = "String form array - $arr `n"
   $message += "Elements in array - " + $arr.count + "`n"
   $message += "Calculation result - $value `n"
   $message += "Created path - /home/$path_name `n"
   $message += "Created file - /home/$path_name/$file_name `n"
   insert_in_file($message)
}

function create_path {
   New-Item -Path "/home" -Name $path_name -ItemType Directory
}

function insert_in_file([string]$message){
   return $message > "/home/$path_name/$file_name"
}

function calc{
 $value = $first_int + $second_int
 create_path
 make_message($value)
}

calc
```

***У него есть явные недостатки: мы игнорируем ошибку в случае, если каталог уже существует, и пытаемся всё равно его создать, а ещё мы не проверяем, переданы ли параметры. Чтобы всё было правильно, добавим проверку в функцию создания каталога, закомментируем переменную, отвечающую за подавление ошибки, и добавим в первую вызываемую функцию проверку на существование параметров:***

```
Param (
[int]$first_int,
[int]$second_int,
[string]$file_name,
[string]$path_name,
[array]$arr
)

#$ErrorActionPreference='silentlycontinue' # Закомментировали подавление

function make_message([int]$value){
    $message = "String form array - $arr `n"
    $message += "Elements in array - " + $arr.count + "`n"
    $message += "Calculation result - $value `n"
    $message += "Created path - /home/$path_name `n"
    $message += "Created file - /home/$path_name/$file_name `n"
    insert_in_file($message)
}

function create_path {
    if ( !( Test-Path -Path "/home/$path_name" ) ) { #Проверка существования каталога
        New-Item -Path "/home" -Name $path_name -ItemType Directory
    }
}

function insert_in_file([string]$message){
    return $message > "/home/$path_name/$file_name"
}

function calc{
  if ( 0 -eq $first_int -or 0 -eq $second_int -or $null -eq $file_name -or $null -eq $path_name -or $null -eq $arr ) { #Проверка существования параметров
      echo "Не все параметры переданы в скрипт"
    exit
  }

  $value = $first_int + $second_int
  create_path
  make_message($value)
}

calc
```

***Разберём по порядку, что произошло:***

- Закомментировали переменную `$ErrorActionPreference='silentlycontinue'`, чтобы не игнорировать ошибку.
- Добавили проверку на пустоту параметров в функции `calc`. В случае, если хотя бы один параметр пуст или равен нулю, скрипт прервёт работу (`exit` — выход из скрипта) и скажет в консоль, что не все параметры переданы. Тут возникает вопрос: почему где-то используется `0 -eq $<переменная>`, а где-то `$null -eq $<переменная>`?
    - Если в скрипт не передать переменную типа `int` (число), то она автоматически принимает значение нуль. Поэтому добавлена проверка равности нулю.
    - Иные типы данных будут равны «пустоте» `$null`.
- Добавили проверку существования каталога в функции `create_path`. Тут возникает вопрос: почему появились дополнительные круглые скобки? Отвечаем: так как для проверки мы используем команду, которая возвращает либо `true`, либо `false`, — мы не можем сравнивать результат с пустотой или нулём. Для того чтобы правильно создать условие «Если результат команды вернул `false`», необходимо команду поместить в дополнительные круглые скобки и перед ними поставить знак отрицания `!`.

***Команда для запуска скрипта будет иметь следующий вид и не будет ничего выводить в консоль:***

```
.\\calc.ps1 -first_int 1 -second_int 11 -arr "Hello","world", "from", "Practicum!" -path_name "practicum" -file_name test.txt
```
## ***Циклы***

*Циклы Bash и PowerShell отличаются только синтаксисом, а функционал, принцип и назначение остаются теми же* 

`while` - пока условие истинно 

Представим, что нам нужно выполнить `ping` десять раз и на одиннадцатую итерацию запустить какую-то программу. Делать это вручную — скучно и долго, как раз для автоматизации «многоходовок» и существуют циклы. Пришло время узнать, как они выглядят в PowerShell, проведя аналогию с Bash, снова взяв примеры из урока про него.

```
#!/bin/bash
#Начинаем отсчёт с 0 присваивая переменной num значение 0
num=0
#Тело цикла проверяет, что это число меньше либо равно 9 (-le)
   while [ $num -le 9 ]
   do
#Теперь пишем тело цикла.
#Сначала выводим число.
      echo $num
#Теперь при помощи expr мы прибавляем к нашему числу 1, а потом записываем полученное число в переменную num
      num=$(expr $num + 1)
   done
```

```
#Начинаем отсчёт с 0 присваивая переменной num значение 0
$num=0
#Тело цикла проверяет, что это число меньше либо равно 9 (-le)
while ( $num -le 9) {
  #Теперь пишем тело цикла.
  #Сначала выводим число.
  echo $num
  #Теперь увеличиваем число на 1, а потом записываем полученное число в переменную num
  $num++
}
```
