## ***Из чего состоит команда и скрипт в PowerShell***

1. ***Командлеты.***
- Вид командлеты  всегда `<глагол>-<существительное>`. Чтобы просмотреть все командлеты, используем команду `Get-Command -CommandType cmdlet` в PowerShell 
![[Pasted image 20260329103050.png]]

2. ***Параметры.***
- Это обязательное или дополнительное значения для выполнения команды или командлеты. У каждого командлеты - свои параметры, увидеть их можно при помощи библиотеки или через `get-help <cmdlet-name>` в консоли 
![[Pasted image 20260329103336.png]]

3. **Операторы.**
- Сюда входят операторы, которые используются для арифметических операций, сравнения и объединения, перенаправлении и  условий

***Арифметические:*** 
- `+` сложение 
- `-` вычитание 
- `/` деление 
- `*` умножение 

***Сравнение:*** 
- `-eq` равно 
- `-gt` больше 
- `-lt` меньше 
- `-contains` содержание 
- `-like` соответствие 

***Перенаправление:*** 
- `>` перезаписать в файл 
- `>>` добавление в файл 
- `>&1` перенаправление в поток success

***Объединение:*** 
- `()` группировка. Чтобы получить нужный объект, мы сгруппировали команду в скобки, а затем использовали свойство `count` для подсчета элементов 

***Условие:*** 
- Логические `И, ИЛИ, НЕТ` 

*Шпаргалка по операторам есть, к примеру, в библиотеке по программированию [BookFlow](https://www.notion.so/2-200-2379f84f02918083b85effa142468d6a?pvs=21).*

4. ***Циклы*** 
- Это методы, которые выполняются только при определенных условиях или для них, а так же перебора массива данных. Мы будем разбирать их подробнее в следующем уроке 

4. ***Переменные*** 
- Некоторые значения, сохраняющие в оперативной памяти компьютера и заданные администратором или системой. Их можно назначать как в скриптах, так и в консоли. Создадим переменную и выведем ее в консоль: 

```
$variable="practicum" 
write-host $variable
```

![[Pasted image 20260329104345.png]]

6. ***Функции*** 
- ***Функция*** - это набор команд, объединенных одним именем и выполняющих некоторую задачу. Функции вызываются по имени, могут принимать параметры и возвращать результат работы. Их удобно использовать в местах, где есть участки повторяющего кода 

***Как выглядит в PowerShell:*** 

```
function Get-DayLog { 
Get-EventLog -LogName Application -Newest 10 | where TimeWritten -ge (Get-Date).AddHours(-14) 
} 
Get-DayLog
```

*Мы создали функцию с именем `Get-DayLog`, которая берёт последние десять событий из логов системы, после чего вызвали её по имени и получили обозначенные события из журнала в консоли.*

![[Pasted image 20260329104804.png]]

7. ***Алиасы*** 
- Это ссылки на команды, файл или каталог. Например, чтобы посмотреть содержимое каталога в PowerShell Windows, можно использовать алиас `ls`, а не `dir` или `Get-ChildItem`
![[Pasted image 20260329105035.png]]

***Зачем нужны алиасы:*** 
- Можно использовать привычный синтаксис, а не командлеты PowerShell
- Командлеты в PowerShell - сложнозапонимаемая сущность, поэтому администраторы часто подстраивают синтаксис под себя  `Get-ItemPropertyValue` наверняка сложнее запомнить, чем собственный алиас вроде `sh val`
- Алиасы могут заменять целые команды 

***Создание и использование алиасы:*** 

```
   new-alias -Name pct -Value "C:\\Program Files\\Cisco Packet Tracer 8.2.1\\bin\\PacketTracer.exe"
   pct
```

*Здесь при назначении алиаса в консоли ничего не выведется, а при его вызове по заданному имени запустится Cisco Packet Tracer.*

8. ***Модули***
- Это набор связанных функций в PowerShell, сгруппированных в блоки: файлов скриптов, сборок и связанных ресурсов 
9. ***Конвейеры (пайпы)***
- Так называют последовательность команд, разделенных символом `|`. В конвейре все команды выполняются последовательно слева направо и передают друг другу объект с прошлой команды 

***Примеры команд для автоматизации***

1. Проверим принимает ли почтовый сервис подключения для получения и отправки писем на стандартном порту: 

```
    Test-NetConnection -ComputerName localhost -Port 25 | Select-Object TcpTestSucceeded
```

![[Pasted image 20260329113030.png]]

- Командлет  `Test-NetConnection` устанавливает соединение с указанным компьютером и портом 
- Параметры `ComputerName` и `Port` указывают, куда и на какой порт подключаться 
- Командлет `Select-Object` выбирает элемент из объекта результата команды, которому передали параметр `TcpTestSucceeded` с кличем элемента 

2. Траблшутинг сервисов, служб и программ начинается с проверки состояния: 

```
Get-Service -Name "Remote*" -Exclude "RemoteAccess" | Select-Object Status | ForEach-Object {$_.Status}
```

![[Pasted image 20260329113351.png]]

- Командлет `Get-Service` отображает все сервисы системы 
- В параметре `Name` указано шаблонное имя, а `Exclude` исключает сервис с именем `RemoteAccess`
- Командлет `Select-Object` выбирает элемент из объекта результата команды, которому передали параметр `Status` с ключом элемента 
- Командлет `ForEach-Object` с параметром `{$_.Status}` перебирает объект и избавляет вывод от хедера

3. Для работы сервисов вроде Kubernetes нужно в определённом месте создать каталог и поместить в него файлы конфигурации. Такие задачи зачастую автоматизируются, и, чтобы случайно не сделать поверх новый каталог и не удалить конфигурацию, ~~сисадмины~~ обычно проверяют наличие нужного файла или каталога по части его имени, например

```
Get-ChildItem | Select-Object Name | Where-Object {$_.Name -like ".kube"} | ForEach-Object {$_.Name}
```

- Командлет `Get-ChildItem` отображает все каталоги и файлы в текущем расположении
- Командлет `Select-Object` выбирает элемент из объекта результата команды, которому передали параметр `Name` с ключом элемента
- Командлет `Where-Object` с параметром `{$_.Name -like ".kube"}` выбирает объект по части имени элемента
- Командлет `ForEach-Object` с параметром `{$_.Name}` перебирает объект и избавляет вывод от хедера
## ***Создание скриптов PowerShell***

- `null` - означает значение всегда равно нулю. Используется для проверки результата выполнения команды или переменной на наличие содержимого 
- `object` - содержит наборы разных типов 
- `bool` - может быть два значения true и false. Используется для проверки на истинность 
- `int` - число 
- `float` - число с плавающей точкой. Нужно для арифметических операций, когда используется не целое число, а дробь или число с запятой или точкой. Так 1.1 нужно указывать в этом типе, так как это дробное число 
- `string` - строка 
- `array` - простой массив `"ключ:значение" ([a, b, c, d, ...])`

***Скрипт PowerShell*** - это файл специального расширения `.PS1`, который содержит набор последовательных команд, условий, циклов и функций 

***Создаем файл `calc.ps1` и пишем скрипт на примере простых математических операций:*** 

```
function calc{
  $val=20 + 30
  return $val
}

calc
```


![[Pasted image 20260408175949.png]]

***Не всегда удобно постоянно менять содержимое скрипта, если нужно сложить другие числа. Здесь можно использовать возможности скриптов и функций, а именно - передача значений:*** 

```
Param (
[int]$a,
[int]$b
)

function calc{
  $val= $a + $b
  return $val
}

calc
```

![[Pasted image 20260408180812.png]]

`Param()` - в нем указываются переменные и назначаются их типы, которые должны быть переданы в скрипт для его работы. Все переменные указываются через запятую 

***Теперь предположим, что нужно вызвать несколько функций подряд и выполнить их по очереди, передавая значения от предыдущей к следующей:*** 

```
Param (
[int]$a,
[int]$b
)

function modern([int]$var){
    return "My first calc with practicum returns - $var"
}

function calc{
  $val= $a + $b
  modern($val)
}

calc
```

![[Pasted image 20260408181244.png]]

*Выполнить математическую операцию и обработать данные, которые скрипт должен получить при его вызове. Например, массив данных, параметры для математических операций и названия каталога и файлов, где будет лежать лог работы скрипта.*

*Задачи нашего скрипта:*

- *Отобразить массив в виде строки в логе. При этом должно быть подсчитано и отражено количество элементов в массиве.*
- *Выполнить математические операции и создать каталог, если его нет.*
- *Вывести результат в файл с перезаписью лога работы с каждым запуском.*

*Также скрипт должен быть логичным и читаемым, а все шаги — разделены функциями и начинаться с математической операции.*

```
Param (
    [int]$first_int,
    [int]$second_int,
    [string]$file_name,
    [string]$path_name,
    [array]$arr
)

$ErrorActionPreference = 'SilentlyContinue'

function make_message {
    param (
        [int]$value
    )

    $message = "String form array - $arr `n"
    $message += "Elements in array - " + $arr.Count + "`n"
    $message += "Calculation result - $value `n"
    $message += "Created path - C:\\$path_name `n"
    $message += "Created file - C:\\$path_name\\$file_name `n"

    insert_in_file -message $message
}

function create_path {
    New-Item -Path "C:\\" -Name $path_name -ItemType Directory | Out-Null
}

function insert_in_file {
    param (
        [string]$message
    )

    $filePath = "C:\\$path_name\\$file_name"
    $message | Out-File -FilePath $filePath -Encoding utf8
}

function calc {
    $value = $first_int + $second_int
    create_path
    make_message -value $value
}

calc
```

***У команды для запуска будет следующий вид и она не будет ничего выводить в консоль:***

```
.\\calc.ps1 -first_int 1 -second_int 11 -arr "Hello","world", "from", "Practicum!" -path_name "practicum" -file_name test.txt
```

***Теперь рассмотрим сам скрипт:***

- С помощью параметра `Param()` мы указали, какие переменные и какого типа передаются в скрипт из консоли.
- Назначили переменную `$ErrorActionPreference= 'silentlycontinue'`. Это системная переменная, которая используется для определения действий при ошибках. В нашем случае скрипт продолжит работу без вывода ошибки в консоль.
- Создали функцию `make_message`, ожидающую получения определённых данных и складывающую (объединяющую) строки в переменную `$message`. В ней:
    - знак `+=` добавляет значения; в нашем случае это сложение строк;
    - спецсимвол `n` означает перенос строки, а с помощью `$arr.count` мы подсчитываем количество элементов строки.
- Создали функцию `create_path` — она создаёт каталог с именем, передающимся в скрипт с помощью параметра `path_name` по пути `C:\\`.
- Создали функцию `insert_in_file`, которая вносит сообщение в файл.
- Создали функцию сложения чисел, параметры которых передаются с помощью параметров `first_int` и `second_int`.

***Ну а теперь углубимся в работу скрипта:***

1. Запуск. При запуске вызывается функция `calc`, которая складывает числа, затем вызывает функцию создания каталога, а после неё — функцию создания сообщения.
2. Создание каталога — функция `create_path`. Каталог создаётся по пути `C:\\` с именем, указанном в параметре `path_name`. Если по этому пути уже есть такой каталог, скрипт выведет в консоль ошибку об этом. А чтобы это сообщение не пугало, мы и назначили переменную `$ErrorActionPreference= 'silentlycontinue'`.
3. Создание сообщения — функция `make_message`.
    - Сначала мы создаём переменную `$message` с помощью знака `=`. В ней будет строка, состоящая из элементов массива, переданного через параметр `arr`.
    - И с помощью символа `+=` добавляем в переменную:
        - строку с количеством элементов массива;
        - cтроку с результатом сложения чисел из функции `calc`;
        - строку с указанием пути созданного каталога;
        - строку с указанием пути файла, в который записано наше сообщение.
    - Вызываем функцию записи в файл.
4. Запись сообщения — функция `insert_in_file`.

На этом этапе созданное предыдущей функцией сообщение записывается в файл.
