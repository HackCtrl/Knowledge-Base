## ***Пайпы***

***Пайп, или конвейер*** - однонаправленный канал меж процессного взаимодействия, который связывает в скриптах несколько команд 

Суть пайпов в том, что результат работы одной программы мы сразу отправляем на обработку другой

***Символ конвейера*** - `|` 
- `команда 1 | команда 2 | команда...N`

***Пример***. Найдем файлы в директории /var/log и посчитаем их количество

1. ***Найдем файлы в директории:*** 
- `find /var/log -type f` 

![[Pasted image 20260307100442.png]]

Получим список файлов во всех доступных папках

1. ***Теперь передадим то, что вывела эта команда, другой команде с помощью символа*** `|`. ***Для подсчета количества строк используем команду*** `wc` ***с аргументом*** `-l`

- `find /var/log -type f | wc -l` 

![[Pasted image 20260307100931.png]]

***Еще один пример использования пайпа*** - одна программа открывает файл, другая вытаскивает из него список, а третья сортирует его в алфавитном порядке 
## ***Первый скрипт***

***Самый простой вариант:*** 
- `mv home/user/script/folder1/file1.txt home/user/script/folder2/`

***Решим данную задачу при помощи пайпа*** - воспользуемся командой cp (копировать), а затем удалим файл из старой директории 

***Перейдём в папку `script` и создадим заготовку для скрипта. Назовём файл [`move.sh`](http://move.sh/) и отредактируем его с помощью редактора `nano`. Расширение `sh` необязательное, но позволяет правильно идентифицировать файлы.***

- `cd /home/user/script` 
- `nano move.sh` 

***И что мы делаем:*** 
```
# копируем файл из одной директории 
cp home/user/script/folder1/file1.txt home/user/script/folder2

# удалем старый файл 
rm home/user/script/folder1/file1.txt
```

***Теперь нужно явно указать, что это не обычный текстовый файл, а скрипт. Для этого добавим в начало строки и укажем  в ней, что скрипт обрабатывает командный интерпретатор:*** 

- `#!` - шебанг - и `/bin/bash` (`#!/bin/bash`)

![[Pasted image 20260307102605.png]]

Сейчас скрипт находится там, где линукс не ожидает увидеть программу, поэтому укажем явно на директорию. Мы находимся в директории /home/user/scrip и именно отсюда запустим скрипт. Текущую директорию можно указать коротко - `.` в начале 

- `./move.sh` 

![[Pasted image 20260307102905.png]]

***Скрипт не сработал, причина ошибки в том, что мы не раздали нужные права:*** 
- `chmod +x move.sh`
### ***Практическое задание***

Напишите скрипт, который переместит папку `/home/..../script/folder1` в папку `folder2` и затем удалит папку `folder1`

***Для удаления папок используется ключ*** `-r`

```
#!/bin/bash
cp -r /home/kali/script/folder1 /home/kali/script/folder2/
rm -r /home/kali/script/folder1
```

## **Улучшаем скрипт**

*В текущем варианте скрипта есть недостаток. Он работает только для папок с конкретными названиями.* ***Сделаем скрипт более универсальным - изменим его так, чтобы он выполнял действия для папок с любым названием*** 

Создадим две переменные `$1` и `$2`: 
```
#!/bin/bash
cp $1 $2 
rm $1 
```

`$1` присваивается папке `folder1`, а `$2` - `folder2`

## ***Переменные окружения***

***Окружения*** - это виртуальная среда, которая создается при запуске консоли. Ее определяет информация о самой консоли и пользователе, который запустил процесс. Переменные окружения помогаю конфигурировать систему и программы. 

***Введем в терминал команду:*** 

- `echo $SHELL`

![[Pasted image 20260309145500.png]]


`$` - обозначение переменной 
`SHELL` - имя переменной 
`/bin/bash` - значение переменной 

*Посмотреть все установленные переменные можно командой* `env` 

***Важная переменная окружения - `PATH`. Команда `echo $PATH` выведет пути, разделенное двоеточием:***
![[Pasted image 20260309150311.png]]

***Список каталогов, в которых выполняется поиск команд:*** 

`pwd` - быстрая команда для того, чтобы узнать в какой папке мы находимся 

![[Pasted image 20260309150512.png]]

`$/bin/pwd` - полный путь к команде 

*Переменная окружения PATH как раз и помогает убрать /bin/. Линукс знает, в какой папке искать команду, и просто выполняет ее* 

***Процесс выполнения команды:*** 
- мы вводим команду - shell ищет эту команду среди встроенных - если находит, то выполняет - если нет, начинает искать в PATH-папках исполняемый файл с таким именем и если находит, то выполняет, если нет - выдает ошибку 

*Именно поэтому мы указываем полный путь к скрипту*

## ***Логические операторы***

## ***Условный оператор `if`***

*ЕСЛИ сегодня 31 декабря, ТО пора праздновать Новый год, ИНАЧЕ — подождём. Запишем в виде скрипта.*

`newyear.sh`: 

```
#!/bin/bash
today=$(date +'%d-%m')
if [ $today = 31-12 ]
  then
  echo 'С Новым годом!'
    else
    echo 'Ждём ещё'
fi
```

![[Pasted image 20260310193707.png]]

1. Создаем переменную `today`, которая присваивает значение команды `date`. Так как `date` выводит день, месяц, год, время, а нам нужны только день и месяц, мы используем конструкцию `+'%d-%m'` (`today=$(date +'%d-%m'`))
2. Затем условный оператор `if` проверяет: `не сегодня ли 31-12` и, если это так, тогда пишет `С новым годом!`, если нет - `Ждем еще`
3. Закрываем конструкцию `fi`
## ***Циклы***

*Например, вы проверяете стабильность пересылки пакетов до сервера. До тех пор, пока программа `ping` не закончит работу, вы не запускаете следующую программу. Здесь пригодится условный оператор, который будет проверять условие не один раз, а много. Когда наконец условие выполнится, он запустит следующее действие.*

***Тело цикла*** - последовательно инструкций, предназначенных для многократного исполнения

***Несколько видов циклов:*** 
- `while`
- `until` 
- `for` 
### ***Цикл `while`***

![[Pasted image 20260310194503.png]]

***Рассмотрим простой пример — выведем числа от 0 до 9. С помощью цикла будем проверять условие «не равно ли число 9». Делать это будем до тех пор, пока оно не окажется верным:*** 

```
#!/bin/bash
#Начинаем отсчёт с 0, присваивая переменной num значение 0.
num=0
#Тело цикла проверяет что это число меньше либо равно 9 (-le).
   while [ $num -le 9 ]
   do
#Теперь пишем тело цикла.
#Сначала выводим число.
      echo $num
#Теперь при помощи expr прибавляем к числу 1, а потом записываем полученное число в переменную num.
      num=$(expr $num + 1)
   done
```

![[Pasted image 20260310194700.png]]

### ***Цикл `until`***

***В цикле `while` требовалось, чтобы условие было истинным и тогда выполнялась задача. Бывает наоборот — цикл должен выполняться, пока условие неверно. Для этого есть цикл `until`:*** 

```
#!/bin/bash
#Начинаем отсчёт с 0, присваивая переменной num значение 0.
num=0
#Тело цикла проверяет что это число не больше 9 (-gt).
   until [ $num -gt 9 ]
do
#Выводит число.
   echo $num
#При помощи expr прибавляем к числу 1, а потом записываем полученное число в переменную num.
   num=$(expr $num + 1)
done
```

![[Pasted image 20260310194835.png]]
### ***Цикл `for`***

***Представьте, что вы работаете в компании с несколькими филиалами. В каждом филиале есть свой сервер, который должен быть доступен 24/7. Простой способ проверить доступность сервера — запустить программу `ping` с указанием адреса сервера. Можно указать адреса по очереди:***

```
ping 192.168.1.2
ping 192.168.2.2
ping 192.168.3.2
ping 192.168.4.2 
```

***Сократим код с помощью цикла:*** 

```
#!/bin/bash
#Задаём переменную "dom" и перечисляем список адресов, которые будем пинговать.
for dom in ya.ru mail.ru vk.com
   do
#Запрашиваем на каждый адрес по 4 запроса.
   ping -c4 $dom
done
```

![[Pasted image 20260310195204.png]]

![[Pasted image 20260310195217.png]]

`-c4` - опция ping означает, что будет отправлено 4 пакета, нужно чтоб не выполнять бесконечный пинг 
### ***Операторы `break` и `continue`***

*Проверка доступности серверов может понадобиться на постоянной основе. В таком случае цикл должен продолжаться до тех пор, пока мы не решим его остановить. Такой цикл называется **бесконечным**.*

`Break` - завершает текущий цикл и передает управление программой команде, которая следует за циклом. Обычно этот оператор завершает цикл при выполнении определённого условия: 

```
#!/bin/bash
i=1
while [ $i -lt 10 ]
   do
   if [ $i -eq 5 ]
      then
      break
   fi
echo "Значение: $i"
   i=$(( $i + 1 ))
done
```

![[Pasted image 20260310195740.png]]

`Continue` - завершает текущую итерацию цикла и начинает следующую, при этом цикл продолжает выполняться: 

```
#!/bin/bash
for (( i = 1; i < 15; i++ ))
do
   if [ $i -gt 5 ] && [ $i -lt 10 ]
   then
   continue
   fi
   echo "Значение: $i"
done
```

## ***Функции***

***Функция*** - это набор команд, объединенных одним именем, которые выполняют определенную задачу 

***Живой пример функции:*** 

```
function home{
   доехать до станции метро "Маяковская"
   подняться на эскалаторе
   выйти на улицу
   дойти до дома
   exit 0
}
```

*Написание функций начинается со слова `function` перед названием или `()` после названия. Так, bash одинаково прочитает `function home` и `home()`. Далее в фигурных скобках перечисляют действия.*

***Функция приветствия:*** 

```
#!/bin/bash
hi() {
echo "Hello world"
}
hi
```
