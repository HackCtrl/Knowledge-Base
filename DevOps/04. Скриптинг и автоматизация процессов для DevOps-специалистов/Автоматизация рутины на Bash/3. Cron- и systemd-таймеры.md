## ***Планировщик cron***

***Cron*** - это утилита, которая позволяет выполнять скрипты в назначенное время и с определенной периодичностью 

*Простая задача, в которой пригодится планировщик, это резервное копирование* 

`Crontab` - это таблица специального формата с расписанием запуска скриптов и программ. Чтобы задания выполнялись, ее нужно отредактировать с помощью команды `crontab -e` 
### ***Синтаксис `crontab`***

***В первых трех линиях таблицы настраиваются ее первичные параметры:*** 
- Сначала необходимо выбрать оболочку для cron. При этом утилита будет стабильно работать в любой из них 
- Если пользователь хочет получить отчет о работе планировщика, нужно указать email 
- Третьей строкой прописывается путь к окружению 

***На практике:*** 

```
SHELL=/bin/bash 
MAILTO=адрес@электроной.почты PATH=/bin:/sbin:/usr/bin:/usr/sbin:/usr/local/bin:/usr/local/sbin
```

***Дальше идут сами задания. Допустим: каждый час в специальный файл записывать вывод команды date:*** 

```
@hourly date > /tmp/date
```

`@hourly` обозначает периодичность раз в час. Есть и другие варианты: 
- `@daily` - раз в день в полночь 
- `@weekly` - раз в неделю 
- `@monthly` - раз в месяц 
- `@yearly` - раз в год 
- `@annualy` - ежегодное обновление в новый год 
- `@reboot` - команда выполнится один раз при перезагрузки системы

***Схема общего синтаксиса `crontab`:*** 

```
* * * * * выполняемая команда
- - - - -
| | | | |
| | | | ----- день недели (0—7) (воскресенье = 0 или 7)
| | | ------- месяц (1—12)
| | --------- день месяца (1—31)
| ----------- час (0—23)
------------- минута (0—59)
```

***Если мы захотим записать дату не каждый час, а каждую минуту, то поменяем запись в crontab на такую:*** 

```
* * * * * date > /tmp/date
```

### ***Задание для самопроверки***

Напишите скрипт, который каждые пять минут будет дописывать в файл фразу `Привет, это скрипт для cron. Сейчас *такое-то* время`

```
#!/bin/bash 
#Файл, куда будет записываться фраза. 
LOGFILE="/tmp/file.txt" 
#Записываем фразу и добавляем дату. echo "Привет, это скрипт для cron. Сейчас: $(date)" >> "$LOGFILE"
```

***После открываем crontab:*** 

```
*/5 * * * * /home/covex/script/script.sh
```

![[Pasted image 20260314093808.png]]

***Ждем 5-10 минут и проверяем:*** 

```
cat /tmp/file.txt
```
### ***Особенности создания расписания***

- Все условия времени запуска проверяются по логическому "И". Но есть и исключения: указанные вместе условия "день недели" и "день месяца" обрабатываются по логическому "ИЛИ", то есть по любому из дней. Такая логика не позволяет создать условия вроде "первый понедельник каждого месяца" или "каждую пятницу 13-го"
- Все файлы crontab должны завершаться пустой строкой 
- Команды файла crontab нельзя разделить на несколько строк 
- Если в автоматически сканируемых каталогов есть файл с точкой в названии, такие файлы будут игнорироваться 
- Синтаксис, используемы для запуска раз в несколько минут со сдвигом, может привести к неожиданным результатам 
- Если символ `%` не экранирован, в команде запуска он переводит строку в команду ъ
## ***Таймеры systemd***

***Systemd-таймеры*** - позволяют запускать сервисы периодически либо при наступлении какого то события 

***В cron содержится и расписание, и действие, а в Systemd эта логика разделена:*** таймер отвечает за управление сервисом, который уже выполняет действие. Для таймеров используются юниты с расширением `.timer`, а для служб - с расширением `.service`. Чтобы управлять сервисом, нужно создать для него таймер

***Создадим юнит, который будет периодически проверять доступность сервера. Начнем с создания файла с расширением `.service` в папке `/etc/systemd/system`:*** 

```
sudo nano /etc/systemd/system/test.service
```

***Начинаем заполнять файл. Первая секция - это [Unit] с описанием службы:*** 

```
[Unit] 
Description=Ping server service
```

![[Pasted image 20260314095856.png]]

*Можно использовать дополнительные возможности systemd. Например, указать зависимости других служб с помощью директивы `[Requires]` или `[Wants]`. Первая более строгая - вторая более мягкая* 

***А если нужно, чтобы служба запускалась при условии, что уже запустилась другая, будем использовать директиву `[After]`. Для юнита, который мы пишем, обязательна уже работающая сеть:*** 

```
After=network.target
```

![[Pasted image 20260314100214.png]]

***Вторая секция - `[Service]`. В ней указывают тип службы - в нашем случае это `oneshot`, подразумевающий единичный запуск - и выполняемое действие:*** 

```
[Service] 
Type=oneshot 
ExecStart=/bin/ping -c4 ya.ru
```

![[Pasted image 20260314100854.png]]

***Завершающая секция - `[Install]` с уровнем запуска:*** 

```
[Install]
WantedBy=multi-user.target
```

`multi-user` - многопользовательский режим без графики 

***Перезапустим службу systemd:*** 

```
systemctl daemon-reload
```

***Запустим сервис:*** 

```
systemctl start test.service
```

***Немного ждем и проверим состояние службы:*** 

```
systemctl status test.service
```

![[Pasted image 20260314102439.png]]

 ***Допустим мы допустили ошибку. Все задачи автоматически логируются - можно посмотреть что случилось:*** 

```
journalctl -u test.service
```

### ***Создаём таймер***

***Название должно соответствовать сервису `test.timer`:***

```
sudo nano /etc/systemd/system/test.timer
```

***Заполним файл. В нем так же должна быть секция `[Unit]` с описанием, а дальше вводим секцию `[Timer]`, в который указываем условия для запуска:*** 

***Таймеры могут быть двух типов*** - событийные и автономные. Первые активируются по событию, а вторые выполняются периодически 

***Событийный таймер - со стартом через некоторое время после запуска системы:*** 

```
[Timer]
OnBootSec=30m
```

![[Pasted image 20260314103927.png]]

***Пример монотонного - `OnCalendar`, который сработает по условию календаря. Формат этому таймеру задается по схеме `DOW YYYY-MM-DD HH:MM:SS`:*** 
- `DOW` — от англ. day of week, то есть «день недели»
- `YYYY` — year, «год»
- `MM` — month, «месяц»
- `DD` — day, «день»
- `HH` — hour, «час»
- `MM` — minutes, «минуты»
- `SS` — seconds, «секунды»

***Все условия можно сочетать так:*** 

```
*:*:00 — каждую минуту 
*-*-2,4,8 *:00:00 — каждый час 2, 4 и 8 числа 
Mon *-*~7/1 — последний понедельник месяца
```

***Можно использовать алиасы, как и в cron: `hourly`, `daily`, `monthly`. Так же мы можем запросить уточнение, корректно ли задача конструкция:*** 

```
systemd-analyze calendar "*-*-2,4,8 *:00:00"
```

![[Pasted image 20260314104651.png]]

***Конечный вариант таймера, который будет запускать проверку работы сервера раз в минуту, выглядит так:***

```
[Unit]
Description=Ping timer
[Timer]
OnCalendar=*:*:00
[Install]
WantedBy=timers.target
```
