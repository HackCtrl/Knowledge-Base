***Автоматизация основана на трех принципов:*** 
- Централизированное управление 
- Масштабируемость 
- Безопасность 
## ***Уровни автоматизации***

1. ***Локальные скрипты на Bash и PowerShell***

Это начальный и самый доступный способ автоматизировать рутинные задачи. Мы пишем скрипт, который выполняет действия последовательно, например делает резервную копию, очищает временные файлы, перезапускает сервис. Скрипты исполняются вручную или через планировщик задач 

2. ***Сценарии с параметрами и логикой*** 

По мере усложнения задач мы добавляем в скрипты условия, циклы, работу с переменными, логирование и обработку ошибок. Скрипт превращается в полноценный инструмент, который можно запускать в разных условиях. На этом уровне обычно подключаются Python или PowerShell-скрипты с расширенной логикой

3. ***Централизированные системы управления*** 

Такие как Ansible. Здесь автоматизация выходит за рамки одного устройства. Мы описываем желаемое состояние сервером и сервисов, и инструмент вроде Ansible сам приводит все в соответствии. Такой подход масштабируется, легче поддерживается и позволяет централизировано управлять  десятками и сотнями узлов 

4. ***Интеграция в CI/CD и оркестрации*** 

На этом уровне автоматизация становится частью общего жизненного цикла разработки и эксплуатации систем. Автоматические проверки, развёртывания, масштабирование — всё это происходит без ручного вмешательства. Используются инструменты вроде Jenkins, GitLab CI, Kubernetes, Terraform и др

***Скрипт*** - дословно переводится как сценарий. Это обычный текстовый файл с последовательностью команд, предназначенных для автоматического выполнения в определенной среде (например, Bash или PowerShell)

***Самый простой скрипт умещается всего в две строки — он выводит на экран фразу «Hello World»:***

```
#!/bin/bash echo " Hello World "
```

***Bash*** - самая популярная командная оболочка Unix-подобных систем, которая выполняет анализ, обработку и выполнение исходного кода или запроса. По другому такие программы называют командными интерпретаторами 

***PowerShell*** - решение для автоматизации задач. Это командная кросс-платформенная оболочка (поддерживается Windows, Linux и MacOS), вобравшая в себя возможности других оболочек. Ещё PowerShell принимает и возвращает объекты .NET — модульной платформы для разработки ПО

```
Write-Host 'Hello, World!'
```

***Python*** - язык программирования

***Масштабируемость*** - один из ключевых принципов автоматизации. Настроенное решение должно легко масштабироваться по мере роста нагрузки 

***Командный интерпретатор*** - программа, которая выполняет анализ, обработку и выполнение исходного кода. Командный интерпретатор читает команды, введенные пользователем, интерпретирует их и выполняет. Он не является аппаратной частью компьютера и не сводится только к оболочке - это самостоятельный инструмент взаимодействия с системой 

*Скрипты записываются в виде обычного текстового файла. Для этого используется любой текстовый редактор — от `nano` и `vim` до VS Code или Notepad++. Скрипт сохраняется с соответствующим расширением (например, `.sh`, `.ps1` или `.py`) и может быть выполнен интерпретатором. Такие файлы не требуют компиляции и запускаются напрямую*
## ***Ansible***

***Ansible*** - это программное решение для удаленного управления конфигурациями, разработанное еще в 2012 году и написанное на Python. Именно с его помощью можно приводить серверы к определенному состоянию - нужно лишь описать, как достичь этого состояния при помощи сценариев. Таким образом, при помощи этой технологии можно быстро переконфигурировать систему, внеся лишь несколько строк в сценарий 

***Почему же именно Ansible?***
- На управляемые узлы не нужно устанавливать дополнительного ПО - все работает через SSH. Это сетевой протокол, который позволяет удаленно управлять ОС 
- Это простая и расширяемая система. Написанное на Python считается простым, значит, и дополнительные модули можно написать без особого труда 
- Прост и язык для написания сценариев 
- Большое количество обновляемой и дополняемой документации 

***Список серверов Ansible можно получить двумя способами:***
1. Из специального текстового файла 
2. С помощью внешнего скрипта, возвращающего нужный список. Для популярных систем мониторинга вроде Zabbix, Cobbler и других есть готовые скрипты 

Ansible подключается у управляемым узлам и собирает информацию о них. На основе этих данных можно использовать разные модули для выполнения сценариев и отдельных задач 

***Вот некоторые из них:***
- Облачные ресурсы и виртуализация: OpenStack и libvirt
- БД: MySQL, PostgreSQL, Redis, Riak
- Файлы: шаблонизация, регулярные выражения и права доступа 
- Мониторинг: Nagios, Monit
- Оповещение о ходе выполнения сценария: Jabber, Irc, почта, MQTT, HipChat
- Сеть и сетевая инфраструктура: OpenStack, Arista
- Управление пакетами: `apt`, `yum`, `rhn-channel`, `npm`, `pacman`, `pip`, `gem`
- Система: LVM, SELinux, ZFS, Cron, файловые системы, сервисы, модули ядра
- Работа с различными утилитами: Git, Hg

Рассмотрим в качестве примера самую простую задачу, которую можем выполнить на группе серверов. Например, проверим, доступны ли они по сети. Самый простой способ сделать это — при помощи утилиты `ping`.

Сперва нужно подготовить специальный файл с именем `hosts` и указать в нём IP-адреса или доменные имена необходимых серверов:

```
$ ansible all -m ping -i hosts 
```

***Теперь обсудим сценарии, которые пишутся на YAML***

***YAML*** - это специальный язык с простым синтаксисом для структурированной записи информации 

***В простых сценариях обязательны два параметра:*** 
- `hosts`  указываются управляемые узлы или группы узлов, к которым нужно применить изменения 
- `tasks`  описывается состояние, в которое нужно привести управляемый узел или же роли 

***Команда для запуска:*** 
- `ansible-playbook <имя_файла_сценария.yml>`
