`WHERE` - при помощи данного оператора, мы можем фильтровать строки 

```
select * from users where ... 
```

- Данным запросом мы говорим, выведи все столбцы из таблицы `users`, где выполняется какое то требование

Оператор `where` выводит все строки, которые удолетворяют какому то условию 

***Пример запроса:*** 
```
select * from users where id = 198;
```

- Выведет только одну строку с `id = 198` 

```
select * from users where id = 198 and True;
```

```
select * from where is_active = 1 and company_id is not null;
```

```
select * from users where referal_user is not null or company_id is not null;
```

- `is not null` - проверяет поле 0 или не 0 
- `or` - работает так, выполняется либо одно либо другое условие, если хотя бы одно из них выполняется, нас это устраивает 
```
select *
from users
where
	referal_user is not null
	and
	company_id is not null
	or
	score > 500
order by score desc;
```

```
select *
from users
where
	(referal_user is not null
	and
	company_id is not null)
	or
	score > 500
order by score desc;
```

- Тут у нас, то что в скобках выполняется одновременно, так как оператор `and` все два условия должны быть истинными. Либо второе условие нас тоже устроит, так как оператор `or`

*Оператор `and` имеет более высокий приоритет чем оператор `or`* 

То есть выше пример, скобки говорят о том, что, то что в скобках, должно выполняться первым и выдавать истину, если же лож, то нас устраивает условие ниже с оператором `or` 

```
select *
from users
where
	(
	referal_user is not null
	)
	or
	(
	company_id is not null
	and
	score > 500
	)
	or
	(
	is_active = 0
	)
order by score desc;
```

- Используем скобки для того, что бы четко видеть условия 

```
select *
from users
where company_id != 7;
```

`!=` - оператор неравенства 

```
select
	is_active,
	is_active + 1 as new_col
from users
were new_col = 2;
```

- Тут мы сталкиваемся с проблемой, оператор `where` нам говорит, что таблицы `new_col` не существует. Это происходит из за того, что оператор `where` отрабатывает быстрее, то есть наш алиас `new_col` присваивается почти в самую последнюю очередь

```
select
	is_active,
	is_active + 1 as new_col
from users
where is_active + 1 = 2;
```
