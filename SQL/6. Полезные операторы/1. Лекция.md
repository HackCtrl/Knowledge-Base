`CONCAT` - оператор, когда несколько строк, нужно объеденить в одну 

`||` - тоже оператор для объеденения строк 

Главное различие данных двух операторов. В случае с `||` если значение `NULL`, то результат оператора тоже будет равен `NULL`. В случае с `concat` даже если будет где то `NULL`, но у нас есть строка в команде, то результат будет все равно строка, даже если мы напишем `NULL`, то все равно будет строка, то есть `concat` по умолчанию использует строку 

`concat_ws()` - оператор, при помощи которого показываем, каким разделителем будем разделять наши дальнейшие строки, и дальше сами строки

```
select
	id,
	username,
	first_name,
	last_name,
	first_name || ' ' || last_name as concat1,
	concat(first_name, null, last_name) as concat2,
	concat_ws(' ', first_name, last_name) as concat3, 
	coalesce(first_name || ' ' || last_name, 'Дорогой клиент') as concat4
from users;
```

`coalesce()` - загружаем в него определенное количество аргументов, и он проверяет значение по данным аргументам столбца, и если там `null,` то идем дальше, если не `null`, то выводим его

*То есть работает так - если в строке что то есть, то он это выводит, если нет, то выводит то, что мы задали в аргументе* 

`nullif()` - это функция, которая сравнивает результат работы первого аргумента и второго аргумента и если они совпадают, то он выводит `null`, если они не совпадают, то выводит 1 аргумент

```
select
	concat_ws(' ', first_name, last_name),
	nullif(concat_ws(' ', first_name, last_name), '')
from users
```

`case when()` - условный оператор, работает как `if else`. Позволяет проверять различные условия и возвращать определенное значение в зависимости от их результата 

```
select *,
	case
		when score < 100 then 'low'
		when score < 500 then 'medium'
		else 'high'
	end as score2
from users;
```
