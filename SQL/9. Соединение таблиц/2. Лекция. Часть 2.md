***Пример синтаксиса***

```
select *
from coderun c
join users u
on c.user_id = u.id
```

`ON` - связывает строки из разных таблиц, он работает в паре с оператором `JOIN`. `ON` задает правило: соединяем строки, если значения в их колонках совпадают

```
select *
from coderun c
right join users u
on c.user_id = u.id
```

Обычный `JOIN` (или `INNER JOIN`) выводит только те строки, где есть совпадения в обеих таблицах. `RIGHT JOIN`выводит _все_ строки из правой таблицы. Если в левой таблице нет совпадения, он все равно покажет строку, заполнив пустые ячейки значением `NULL` (пустота)

Если дописать условие в код выше `where c.id is null`, то это покажет разницу, что `join` просто выведет пустую таблицу, а `right join` выведет все строки из правой таблицы и покажет пустые значение при помощи `NULL`

*Так же при помощи алиасов, мы можем указывать, что именно хотим вывести*

```
select c.*, u.username
from coderun c
join users u
on c.user_id = u.id
```

```
select *
from users u
left join coderun c
on c.user_id = u.id
where c.id is null
```

*Так же мы можем комбинировать условия* 

```
select *
from coderun c
join users u
on c.user_id = u.id
join problem p
on p.id = c.problem_id
```

```
select *
from coderun c
right join users u
on c.user_id = u.id
join problem p
on p.id = c.problem_id
```

```
select *
from coderun c
right join users u
on c.user_id = u.id
left join problem p
on p.id = c.problem_id
where c.problem_id is null
```

*Самое главное правильно комбинировать и использовать типы `join`, иначе могут возникнуть большие проблемы работы с данными в таблицах*
