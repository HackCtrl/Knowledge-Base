# ***Общие сведения***

*Понятие "проникновение" здесь относится к получению несанкционированного доступа к системе и не имеет отношения к "тесту ручки" или аналогичным искаженным интерпретациям*

***Пентест представляет собой набор мероприятий, которые создают сценарий, имитирующий реальную атаку на ИС или приложения, и включает в себя действия на следующие элементы:*** 
- Сети 
- Приложения и ПО 
- Устройства 
- Физические системы
- Человеческий фактор 

![[Pasted image 20260315095836.png]]

***Согласно ГОСТ Р 58 143-2018 «Информационная технология. Методы и средства обеспечения безопасности. Детализация анализа уязвимостей программного обеспечения в соответствии с ГОСТ Р ИСО/МЭК 15 408 и ГОСТ Р ИСО/МЭК 18 045. Часть 2. Тестирование проникновения», определяются следующие разделы проведения тестирования на проникновение:***
- Планирования тестирования на проникновения 
- Разработка тестов 
- Проведение тестирования на проникновение 
- Формирование отчетности по результатам тестирования на проникновение 


***Внешний:*** 
- Сбор информации 
- Поиск уязвимостей 
- Эксплуатация уязвимостей 

***Внутренний:*** 
- Поиск доступной информации 
- Повышение прав доступа за счет эксплуатации 
- Повторный поиск информации с повышенными правами 
![[Pasted image 20260315100129.png]]

***Внешний пентест*** - моделирует действия злоумышленника с наружи: 
- Первый этап - разведка 
	- Инструмент - nslookup, dig или SamSpade 
- Второй этап - сканирование (исследуется топология сети, сканирование открытых портов)
	- Инструменты - NMap, Nessus, XSpider, OpenVAS, Maxpatrol 
		- Цель - определение служб, запущенных на целевых хостах (тип используемой ОС, проверка на наличие уязвимостей)
- Третий этап - получение доступа (поддержание доступа осуществляется путем установки бэкдоров, которые позволяют исследователю безопасности повторно подключаться к системе)

Отдельное направление внешнего пентеста - социальная инженерия 

***Внутренний пентест*** - проводится в целях проверки инфраструктуры заказчика на наличие уязвимостей изнутри 
- Первый этап - разведка (поиск доступной информации)
- Второй этап - получение доступа (повышение прав доступа за счет эксплуатации известных уязвимостей)
- Третий этап - повторный поиск информации с повышенными правами (в зависимости от согласованного с заказчиком ТЗ)
- Четвертый этап - анализ результатов 

***Пентесты включают в себя как автоматизированные процедуры, так и ручное тестирование:*** 
1. Ручное тестирование - это тестирование, при котором тестировщик вручную ищет уязвимости в ПО. Это можно делать через перебор паролей, анализа защиты периметра сети и социальную инженерию 
2. При автоматизированном тестировании применяются специальные инструменты для автоматической проверки системы на наличие уязвимостей 
3. Сочетание ручного и автоматизированного тестирования, при кортом применяют как ручные так и автоматизированные методы. Так можно получить более полную картину об уровне безопасности 
# ***Методики тестирования на проникновение***

***Существуют несколько известных и авторитетных методик и стандартов для проведения тестирования на проникновение:*** 
- OSSTMM
- NIST SP 800-115
- OWASP
- ISSAF
- PTES

***OSSTMM (The Open Source Security Testing Methodology Manual)*** - это один из наиболее популярных стандартов, разработанный *Institute for Security and Open Methodologies (ISECOM)*. Этот стандарт является универсальным руководством для проведения тестирования на проникновение, и его применение может служит основой для таких испытаний 

***Метология OSSTMM предлагает пять основных направлений для тестирования операционной безопасности:*** 
- Безопасность человека 
- Физическая безопасность 
- Беспроводная связь 
- Телекоммуникации 
- Сети передачи данных 

***NIST SP 800-115*** - является одной из обязательных метологий для проведения пентестов, и ее применяют профессиональные аудиторские компании для обеспечения безопасности ИС и данных организаций. Этот стандарт представляет собой техническое руководство, которое может быть применено для оценки уровня ИБ организаций в различных сферах 

***OWASP (Open Web Application Security Project)*** - это открытое интернет сообщество, которое предоставляет наиболее полную метологию для проведения тестирования безопасности веб-приложений, веб-сайтов и API. Документация, представляемая OWASP, является ценным ресурсом для любой ИТ-компании, которая стремится создать безопасное ПО 

*Одним из преимуществ OWASP является то, что их метология охватывает тестирование на каждом этапе жизненного цикла разработки приложения* 

***ISSAF (Information System Security Assessment Framework)*** - был разработан Open Information Systems Security Group (OISSG). Этот документ охватывает множества аспектов, связанных с ИБ, и представляет подробные рекомендации по проведению тестирования на проникновение 

***PTES (Penetration Testing Methodologies and Standards)*** - представляет рекомендации для проведения базового тестирования на проникновение, а так же предлагает несколько расширенных вариантов тестирования, которые подходят для организаций с более высокими требованиями к ИБ. Одним из основных преимуществ PTES является то, что он предоставляет подробно описание целей и ожиданий от проведения пентеста 

***Основные этапы PTES включают в себя:***
- Сбор информации 
- Моделирование угроз 
- Анализ уязвимостей 
- Эксплуатация уязвимостей 
- Составление отчета 
# ***Как проводят тестирование: виды пентестов***

*Как и злоумышленник, изначально исследователь может быть ограничен в информации об информационной системе. Для решения этой проблемы существует три основных метода к проведению пентестов: белый, черный и серый ящики. Отличаются они по количеству информации о системе, которая есть у пентестера перед началом процедуры.*
### ***Черный ящик***

*Характеризуется тем, что пентестер не обладает предварительной информацией о системе и действует так, как если бы он был злоумышленником, которые впервые столкнулся с данной системой. При этом пентестер имеет доступ только к данным, которые находятся в открытом доступе. Эта методика часто используется реальными хакерами* 

***Плюсы:*** 
- Эмулирует реальные условия атаки, когда хакер не имеет знаний о системе 
- Поваляет оценить уровень защиты системы внешними атакующими 
- Независимое и объективное тестирование без предварительного знания системы 

***Минусы:*** 
- Ограниченное понимание внутренний структуры и деталей системы 
- Риск упустить специфические уязвимости, которые требуют знания системы 
- Возможны ложные срабатывания, не связанные с уязвимостями 

![[Pasted image 20260315105148.png]]
### ***Белый ящик***

*Пентестер получает полную информацию о системе: устройства сети, средства защиты, исходный код, процессы и так далее. При этом все сотрудники должны быть осведомлены о том, что проводится тестирование* 

***Плюсы:*** 
- Полный доступ к внутренней структуре и дизайну системы 
- Тестирование системы на всех уровнях, включая приложения, сети и аппаратное обеспечение 
- Детальное понимание системы и возможность оптимизации безопасности 

***Минусы:*** 
- Требует активного участия со стороны заказчика 
- Занимает много времени 
- Высокие затраты на подготовку и реализацию 

![[Pasted image 20260315105508.png]]
### ***Серый ящик***

![[Pasted image 20260315105542.png]]

*Эмуляция действий нарушителя с ограниченными знаниями о системе*

***Плюсы:*** 
- Комбинирует преимущества исследования с доступными знаниями о системе 
- Позволяет проверить систему с различных уровней доступа 
- Более реалистичное моделирование атаки с учетом частичной информации о системе 

***Минусы:*** 
- Возможно не полное понимание системы и упущение специфических уязвимостей 
- Затруднение при определении объема информации, доступной для тестирования 
- Требует дополнительного согласования и сотрудничества со стороны клиента
