#### 1. Необходимое оборудование (стартовый набор)

Для базовых экспериментов нам понадобится классический набор компонентов:

- **Микроконтроллер:** Плата _Arduino UNO_ (сердце проекта на базе чипа ATmega328P).
- **Периферия:**
    - Светодиод (может быть встроенным в плату, но для внешних цепей берем отдельный).
    - Резистор (номинал 220 Ом — стандарт для ограничения тока через светодиод, чтобы его не сжечь).
    - Кнопка (тактовая, для дискретного ввода сигнала).

#### 2. Структура скетча (базовой программы)

Любая программа для Arduino (скетч) строится вокруг двух обязательных функций. Это не просто функции, это управляющие хуки (hook functions) среды:

- **`void setup()`**
    - **Назначение:** Выполняется **один раз** при старте микроконтроллера (после подачи питания или нажатия кнопки Reset).
    - **Задачи:** Сюда выносится вся инициализация: настройка скорости обмена по Serial-порту (`Serial.begin(9600)`), определение режимов работы пинов (вход/выход) через `pinMode()`, запуск различных библиотек. По сути, это "конструктор" нашей программы.
- **`void loop()`**
    - **Назначение:** Бесконечный цикл. Содержит основной алгоритм работы устройства.
    - **Особенность:** После выполнения последней инструкции в `loop()` программа автоматически переходит на первую строку этой же функции и выполняет всё заново. Пока есть питание, loop будет крутиться.
#### 3. Управляющие конструкции (нюансы)

В целом, конструкции стандартные для C++ (условия, циклы). Лектор акцентирует внимание на специфике работы `switch-case`:
- **`break`** — критически важная команда внутри `case`. Если забыть её поставить, выполнение программы "провалится" дальше (механизм _fall-through_) и начнет выполнять код из следующего блока `case`, что часто приводит к логическим ошибкам.
- **Применение:** Конструкция `switch-case` с `break` очень удобна для реализации конечных автоматов, обработки пунктов меню или различных состояний устройства (режим ожидания, работа, настройка).
#### 4. Комментарии и директивы препроцессора

Это инструкции для компилятора, которые обрабатываются до начала компиляции кода. Их принято писать в самом начале скетча (глобальная область).
- **`#define`** — директива для создания макроса. По сути, это текстовый "поиск и замена" в коде до компиляции. Например, `#define LED_PIN 13`. Компилятор везде подставит число 13 вместо `LED_PIN`. Память под это не выделяется (в отличие от переменной).
- **`#include`** — директива, которая буквально вставляет содержимое указанного файла (чаще всего заголовочного файла библиотеки) в наш скетч. Это нужно для подключения готовых библиотек для работы с датчиками, экранами и т.д.
#### 5. Типы данных и операторы

В презентации упомянут конкретный микроконтроллер — **ATmega328P** (стоит на Arduino UNO). Для него важно помнить о размерах типов данных, так как оперативной памяти там мало (всего 2 КБ).
- **Типы данных:** Стандартные для C++ (int, float, char, byte), но нужно помнить, что `int` здесь занимает 2 байта (от -32768 до 32767).
- **Операторы:** Символы для выполнения действий над переменными (арифметические `+`, `-`, логические `&&`, `||`, сравнения `==`, `!=` и т.д.). Пункт дается как вводный, подразумевается, что студент с ними знаком.
