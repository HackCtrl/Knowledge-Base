**🔥 ИНЦИДЕНТ НЕДЕЛИ**

**Китайские хакеры атаковали VMware ESXi через уязвимости нулевого дня**

В декабре 2025 года компания Huntress обнаружила целенаправленную атаку на виртуальные машины VMware ESXi. Атакующие использовали три уязвимости нулевого дня, которые существовали ещё до их официального обнародования в марте 2025 года.

**🔍 Что использовали хакеры:**

- **Уязвимости:** CVE-2025-22224 (9.3 CVSS), CVE-2025-22225 (8.2 CVSS), CVE-2025-22226 (7.1 CVSS)
    
- **Вектор атаки:** Взломанный VPN-шлюз SonicWall как точка входа
    
- **Цель:** Полный контроль над гипервизором ESXi из гостевой виртуальной машины
    

**⚙️ Технические детали атаки:**

1. **Эксплойт MAESTRO** определял версию ESXi и использовал уязвимости для записи полезной нагрузки в память VMX
    
2. **Неподписанный драйвер MyDriver.sys** загружался через утилиту KDU
    
3. **Бэкдор VSOCKpuppet** устанавливался на порт 10000 VSOCK для скрытого доступа
    
4. **Плагин GetShell** позволял выполнять команды и передавать файлы между виртуальными машинами и гипервизором
    

**🛡️ Что делать, если используете VMware:**

- ✅ Немедленно обновите ESXi до последней версии
    
- ✅ Проверьте логи на наличие devcon.exe и MyDriver.sys
    
- ✅ Мониторьте необычную активность на порту 10000 (VSOCK)
    
- ✅ Рассмотрите возможность отключения ненужных служб VMCI и HGFS
    
- ✅ Проверьте VPN-устройства SonicWall на наличие компрометации
    

**🎯 Почему это важно:**  
Атака полностью обходит изоляцию виртуальных машин. VSOCK-коммуникация не отслеживается стандартными сетевыми средствами защиты. Угроза особенно опасна для компаний, использующих виртуализацию для критической инфраструктуры.

### **⚠️ ТРЕНДОВАЯ УГРОЗА**

**Уязвимость в умных инвалидных колясках позволяет взламывать их по Bluetooth без пароля**

Исследователи обнаружили критическую уязвимость в электрических инвалидных колясках WHILL Model C2 и Model F. Проблема позволяет злоумышленнику в радиусе Bluetooth получить полный контроль над устройством без каких-либо учетных данных.

**🔍 Суть уязвимости:**

- **CVE-2025-14346** — критическая уязвимость (CVSS оценка не указана, но классифицирована CISA как критическая)
    
- **Проблема:** Полное отсутствие аутентификации в Bluetooth-соединении
    
- **Устройства:** WHILL Model C2 и Model F (персональные электрические инвалидные коляски)
    

**⚙️ Что может злоумышленник:**

1. Подключиться к коляске по Bluetooth без пароля
    
2. Управлять движением (вперед/назад, повороты)
    
3. Отключать ограничители скорости
    
4. Менять профили настроек
    
5. Теоретически — сохранять контроль даже после выхода из зоны действия Bluetooth
    

**🛡️ Меры защиты:**

- ✅ Проверить наличие обновления прошивки от WHILL (выпущено в декабре 2025)
    
- ✅ Отключить Bluetooth, когда он не используется
    
- ✅ Владельцам — связаться с WHILL для подтверждения установки патча
    
- ✅ Разработчикам IoT-устройств — обязательно внедрять аутентификацию даже для Bluetooth Low Energy
    

**🎯 Почему это тревожно:**

1. **Физическая опасность:** В демонстрации коляску дистанционно сбрасывали с лестницы
    
2. **Медицинский контекст:** Устройство имеет разрешение FDA, но не имело базовых мер защиты
    
3. **Тренд IoT:** Очередной пример, когда безопасность "умных" устройств отстает от их функциональности
    
4. **Сложность обновления:** Неясно, как патч доставляется пользователям — автоматически или вручную
    

**📌 Источники:** CISA Advisory, исследование QED Secure Solutions, демонстрация SecurityWeek

**🛠️ ИНСТРУМЕНТ В ДЕЛЕ**

**BurpGPT Pro: Эволюция пассивного сканирования с помощью AI**

Команда проекта BurpGPT выпустила коммерческую версию расширения для Burp Suite, которая интегрирует GPT-модели в процесс анализа безопасности. Инструмент перешёл из бета-статуса в профессиональное решение для пентестеров и security-исследователей.

**🔧 Техническая суть:**

- Пассивный сканер отправляет HTTP-трафик в выбранную GPT-модель через OpenAI API
    
- Использует систему плейсхолдеров (`{REQUEST}`, `{URL}`, `{RESPONSE_BODY}`, `{METHOD}`) для контекстного анализа
    
- Поддерживает тонкую настройку лимитов токенов и выбор модели
    

**⚡ Ключевые функции Pro-версии:**

- Поддержка локальных AI-моделей (повышение конфиденциальности)
    
- Персистентное хранение конфигураций
    
- Улучшенная генерация отчётов в Burp-интерфейсе
    
- Точный контроль максимального количества токенов
    

**⚠️ Критически важные ограничения:**

- **Передача данных в OpenAI** — неприемлемо для систем с повышенными требованиями к конфиденциальности
    
- **Зависимость от качества промптов** — эффективность напрямую определяется навыками оператора
    
- **Обязательная постобработка** — высокий процент ложных срабатываний требует экспертной верификации
    

**📋 Системные требования:**

- Burp Suite Professional/Community ≥ 2023.3.2
    
- JDK ≥ 11
    
- OpenAI API ключ (для облачной версии)
    

**🎯 Практическое применение:**  
Инструмент эффективен для:

- Поиска логических уязвимостей, недетектируемых сигнатурными методами
    
- Анализа сложных SPA-приложений и API
    
- Генерации гипотез для дальнейшего manual-тестирования
